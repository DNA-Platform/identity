### [Coding Policy](../coding-policy/.cover.md)

**Read when:** you're about to write or review $Chemistry code.

Libby: Three chapters covering the conventions specific to this codebase. Chapter 1 (the $ convention): `$` prefix separates intrinsic identity from extrinsic context — `$Chemical` is the framework concept, `$()` creates the React wrapper, `$derivatives$` is framework-internal. camelCase always, no ALL_CAPS. Chapter 2 (reactive patterns): scope-tracked reactivity through getters/setters, view purity (same state → same output), safe composition (chemicals compose freely because views are pure), the getter pattern for extensibility (`get Card() { return DefaultCard; }`). Chapter 3 (framework vs application): `chemistry/src/` is the framework (Cathy's territory, maximum test coverage), `chemistry/app/` is the Lab (Phillip's territory, dogfooding), `.public/` is the public face (the teaser page, deployed to GitHub Pages).

## The field guide

**Read when:** you're creating or modifying library content — writing a book, adding a chapter, fixing links.

0. [The library](00-the-library.md) — the two-layer model: objective (shared, normative) at the top, subjective (personal, earned) under `..team/`. The direction rule: subjective links to objective, never the reverse.
1. [Anatomy of a book](01-anatomy-of-a-book.md) — a book is a directory with `.cover.md` (YAML frontmatter: title, author, summary, links) and numbered chapters (`NN-slug.md`). The cover carries the TOC.
2. [The linking garden](02-the-linking-garden.md) — three kinds of links: citations (reference-style, at the bottom), inline (in prose), and cross-references (frontmatter `links:`). Broken citations are weeds.
3. [Growth and refactoring](03-growth-and-refactoring.md) — chapters become books when they need their own TOC. Books become anthologies when chapters become books. Don't split linear content just for length.
4. [Subjects and catalogues](04-subjects-and-catalogues.md) — a catalogue organizes books into a subject. Its cover describes relationships and reading order.
5. [Authorship and autobiography](05-authorship-and-autobiography.md) — every book has an author link to an autobiography. Autobiographies self-link: `author: "[Name](.cover.md)"`. The autobiography defines the author, not the other way around.
6. [Academic papers as books](06-academic-papers-as-books.md) — convention for turning published papers into navigable library books. Provenance tags, figure chapters, the PDF as authoritative source.
7. [Voice and nametags](07-voice-and-nametags.md) — graduated to [Protocols ch. 1](../protocols/01-voice-and-nametags.md). See there for the full convention.
