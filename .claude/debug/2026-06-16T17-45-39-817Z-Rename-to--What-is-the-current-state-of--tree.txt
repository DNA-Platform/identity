ControlType.Text | The real caveats, which haven't gone away:
ControlType.Text | Native OS-level UI is the main gap. Playwright does not intercept the native Electron dialog API (showOpenDialog, showSaveDialog, showMessageBox, etc.) because those calls happen in the main process and go straight to OS APIs.
ControlType.Text |  The recommended workaround is to use electronApplication.evaluate() to replace those methods in the main process so tests run deterministically without any OS-level UI
ControlType.Text |  — e.g. stubbing the open dialog to return a fixed path.
