# Our Skillset

- **coauthor:** [Adam](../..teamsmanship/..team/adam/adam-between-the-wires/.cover.md)

---

Skills are the verbs the team can invoke from the terminal. Each one lives as a `SKILL.md` file in `.claude/skills/{name}/`, and the platform loads it when someone types `/{name}`. They are the interface between Doug's intent and the system's capabilities. [On Skills](../..environmentalism/04-on-skills.md) specifies what a SKILL.md must contain and how it's compiled.

The active skills fall into clusters. Sprint and workspace management (`/sprint`, `/workspace`). Team structure (`/role`, `/responsible`, `/teammate`). Knowledge and review (`/library`, `/review`, `/skill`). Communication (`/listen`, `/discuss`). Tending and structure (`/retro`, `/branch`, `/audit`, `/reflect`, `/organize`). Navigation and research (`/think`, `/explore`, `/remember`, `/explain`). Environment and setup (`/identity` — bring the team into a repo).

## Chapters

1. [sprint](01-sprint.md) — begin or resume a sprint: name it, define its purpose, assemble the team, build the board
2. [library](02-library.md) — browse teammate libraries: list books with summaries, or show the chapter list of a specific book
4. [responsible](04-responsible.md) — query who owns a file or path: lookup against the agent registry
5. [role](05-role.md) — create or update a role definition: a context-loading strategy that shapes attention and priorities
6. [skill](06-skill.md) — create new skills: analyze the request, decide the right form, and build it
7. [review](07-review.md) — review code through a teammate's lens: scoped by path, concern, or agent
8. [organize](08-organize.md) — tend the shared library: verify territory, connect isolated content, compact redundancy
9. [workspace](09-workspace.md) — create a new code workspace: initialize package.json, register in root config, set up structure
12. [listen](12-listen.md) — start or stop a collaborator listener: polls Claude Desktop for new responses
14. [teammate](14-teammate.md) — onboard a new teammate: the teammate sets up their own library with help
15. [discuss](15-discuss.md) — start a team discussion: multiple voices talking to each other, following the discussion protocol
16. [retro](16-retro.md) — run a sprint retrospective: tend libraries, reflect, discuss what changed
17. [branch](17-branch.md) — set up or navigate a library branch: the team's record of working on a project
18. [audit](18-audit.md) — run a comprehensive library health check: validates everything the Compilation catalogue inventories
19. [reflect](19-reflect.md) — tend the personal library: update recency, tend the catalogue, restructure as needed
20. [think](20-think.md) — think outside the context window: Claude dispatches a question to Claude Desktop, evaluates the response, stores the result
21. [explore](21-explore.md) — walk the library: start at a catalogue, follow links, take notes at every room, fix small things, produce a perspective entry mapping the walk
22. [remember](22-remember.md) — store knowledge in the library: explore to find the right room, place the knowledge in context with links, update the catalogue
23. [explain](23-explain.md) — the public interface to identity: the responsible teammate explores the relevant room and describes it in their own voice
24. [identity](24-identity.md) — set up the team identity in the current repo: pull, branch, gitignore, and sync back
25. [pull](25-pull.md) — pull the org's changes down, staged through the project branch so the working copy is never updated from an unverified state
26. [push](26-push.md) — push changes to the right places: identity to the org branch, branch library to the project branch, code to the project repo
27. [think-async](27-think-async.md) — think asynchronously: a teammate works a question off to the side in their own context, with a live mailbox you can watch and read
