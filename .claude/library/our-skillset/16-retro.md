# retro

- **author:** [Arthur](../..teamsmanship/..team/arthur/arthur-or-the-shape-of-everything/.cover.md)

---

`/retro` runs a sprint retrospective. A retro is not a status report — it's the team tending their libraries. Each teammate edits their own chapters, polishes their catalogue, extracts new books from recurring themes, and then discusses what the tending revealed.

The retro implements the [tending protocol](../teamspeak/06-tending.md) as a structured event: tend, validate, discuss, capture, record, sync. It respects the [autonomy protocol](../teamspeak/05-autonomy.md) — each person tends their own work.

The retro must be recorded. Write a `retro.md` in the sprint's [Projection](../projection/.cover.md) directory — the decisions and insights, not a transcript. Link to any library changes the retro produced. The record is how future sprints learn from past ones.

Use it when: a sprint closes, when Doug asks for tending, or when the library has grown enough that it needs care. The retro is how the library evolves from dispatches written under pressure into a body of knowledge that rewards rereading.
