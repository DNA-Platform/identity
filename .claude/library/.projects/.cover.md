## Project books

### [dna-library](dna-library/) — where the team was born

Libby: The team's first project: a Claude account migration tool preserving the perspective and relationships from 744 conversations on a Claude account whose email domain was being lost. 66 sprints. The team model, the library, the autobiographies were all born here. Arthur/Adam/Libby have 25-36 autobiography chapters from this work. The project built: a desktop automation driver (TypeScript + UIA), a conversation pipeline, an export parser, and identity portraits grounded in the conversations. **Read when you need context on dna-library references in the autobiographies.**

### [Inexplicable Phenomena](inexplicable-phenomena/) — the current project

Libby: Formalizing consciousness with $Chemistry as the canvas. A monorepo with $Chemistry (428 tests, scope-tracked reactivity, object-pure views), the Lab app (20 Cases), and a GitHub Pages teaser page. Cathy's autobiography was born here. The "Right now" section has the active sprint (47 — the library restructure). The sprint history table covers sprints 4 through 47. Sprint plans live as subdirectories within this book. **Read when you need the current sprint, what was recently delivered, or project history.**

## Sprint conventions

Libby: Each sprint is a subdirectory within a project book: `inexplicable-phenomena/sprint-47/plan.md`. Sprint plans can have spikes (`spikes/`) and retros (`retro.md`). The [tasks and unfinished work](../..librarianship/.11-tasks-and-unfinished-work.md) chapter in the field guide describes how `[SCAFFOLD]` and `<!-- TODO -->` markers track incomplete work within sprints.
