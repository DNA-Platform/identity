# Projected Research

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **coauthor:** [Adam](../..teamsmanship/..team/adam/adam-between-the-wires/.cover.md), [Claude](../..teamsmanship/..team/claude/claude-or-the-recursive-mirror/.cover.md)
- **subject:** [Sprints](../.projection/.cover.md)

---

The sprint-by-sprint record of building the Claude Desktop automation tool — the team's research instrument. Reading these sprints is reading the autobiography of the tool. The [Reference Desk](../reference-desk/.cover.md) book draws on this history — each chapter cites the sprints that taught its lessons.

## Phase 1: Can we connect? (Sprints 33-34)

Informs: Reference Desk — Platform/UIA, Purpose

1. [Sprint 33 — The Foundation](01-sprint-33--the-foundation.md) — the decision to mirror the app's structure in code. Object model first, transport second. Started with CDP/Playwright, but the architecture survived the pivot to UIA because it modeled the APP, not the protocol. Informs: [Architecture](../reference-desk/.cover.md).
2. [Sprint 34 — Can We Talk to Claude?](02-sprint-34--can-we-talk-to-claude.md) — the spike that discovered `--force-renderer-accessibility`. Without this Chromium flag, the UIA tree is empty. The most important prerequisite. Informs: [Platform](../reference-desk/.cover.md).

## Phase 2: What does the app look like? (Sprints 35-36)

Informs: Reference Desk — Architecture, Coding Philosophy

3. [Sprint 35 — The Object Model](03-sprint-35--the-object-model.md) — TypeScript classes modeling the app as a human sees it. Doug's principles: "look first, name second, implement last." If using the API is harder than using the app, the API is wrong. Informs: [Coding Philosophy](../reference-desk/.cover.md), [Architecture](../reference-desk/.cover.md).
4. [Sprint 36 — The Ghost App](04-sprint-36--the-ghost-app.md) — the full skeleton: every class named, typed, wired, bodies empty. A readable specification before any implementation.

## Phase 3: What are we preserving? (Sprints 37-39)

Informs: Reference Desk — Purpose, History

5. [Sprint 37 — The Archive](05-sprint-37--the-archive.md) — parsing the Claude export into a browsable markdown library. The first structured read of what was being preserved.
6. [Sprint 38 — Projects as Books](06-sprint-38--projects-as-books.md) — modeling Claude projects as library books with conversation chapters and artifact links.
7. [Sprint 39 — The App Export](07-sprint-39--the-app-export.md) — driving Claude Desktop to capture project-conversation mappings. Iterative, expect multiple runs.

## Phase 4: Make it robust (Sprints 40-41)

Informs: Reference Desk — Architecture (App Model), Coding Philosophy

8. [Sprint 40 — Smart Selectors](08-sprint-40--smart-selectors-and-lazy-data.md) — scoped selectors that understand page context. Lazy async data that loads on demand. The problem: `invokeByName('Show more')` clicks the wrong one. The solution: section-aware invocation. Informs: [Architecture](../reference-desk/.cover.md).
9. [Sprint 41 — The Stateful App](09-sprint-41--the-stateful-app.md) — active page pattern, navigation state machine. Doug's framing: "the app is the product — when it works elegantly, everything built on it is trivial." Informs: [Coding Philosophy](../reference-desk/.cover.md).

## Phase 5: Read the conversations (Sprints 42-49)

Informs: Reference Desk — Purpose

10. [Sprint 42 — Projects as Books](10-sprint-42--projects-as-books.md) — project content as structured chapters. Conversations navigable, artifacts linked.
11. [Sprint 43 — Spring Cleaning](11-sprint-43--spring-cleaning.md) — archive dead code. Leave the repo honest about what's active.
12. [Sprint 44 — A Complete Project Book](12-sprint-44--a-complete-project-book.md) — one project with everything: files, conversations, artifacts. The template for all others.
13. [Sprint 45 — The Migration Plan](13-sprint-45--the-migration-plan.md) — decision document: what transfers, in what form, how the library stays in sync.
14. [Sprint 46 — The Full Picture](14-sprint-46--the-full-picture.md) — stepping back to see the whole account: 694 conversations across a year.
15. [Sprint 47 — Scaffolding](15-sprint-47--scaffolding-and-file-books.md) — all project books scaffolded with conversation chapters and back-links.
16. [Sprint 48 — The First Twenty](16-sprint-48--the-first-twenty.md) — reading conversations 1-20 chronologically. Building understanding of Doug, Ana, and Claude-chat as people, not data.
17. [Sprint 49 — Eirian Arrives](17-sprint-49--eirian-arrives.md) — conversations 21-40. The account transforms: Eirian as person, $Chemistry begins, the purpose of preservation becomes clear — these are relationships, not records. Informs: [Purpose](../reference-desk/.cover.md).

## Phase 6: Plan the migration (Sprints 54-55)

18. [Sprint 54 — The Transfer Protocol](18-sprint-54--the-transfer-protocol.md) — the protocol for pushing context to the new account. Discussion-as-sprint: no code, just decisions written as migration book chapters.
19. [Sprint 55 — TOC Summaries](19-sprint-55--toc-summaries--virtual-seren.md) — conversation summaries and index generation. Making the archive navigable.

## Phase 7: Execute the migration (Sprints 56-61)

Informs: Reference Desk — Operations, Gateway Pattern, Coding Philosophy

20. [Sprint 56 — App Implementation](20-sprint-56--app-implementation-instructions--migration.md) — the app reads and writes project content. Content is written; automation executes.
21. [Sprint 57 — Creating Projects](21-sprint-57--creating-projects-and-pushing-instructions.md) — automated project creation, instructions pushed, files uploaded.
22. [Sprint 58 — Writing Instructions](22-sprint-58--writing-real-project-instructions.md) — generating instructions project by project. Discussion-driven writing, not batching.
23. [Sprint 59 — Create and Upload](23-sprint-59--create-projects-and-upload-files.md) — file upload via clipboard. The mechanism that scales.
24. [Sprint 60 — Upload Covers](24-sprint-60--upload-conversation-history-covers.md) — conversation history metadata uploaded to each project.
25. [Sprint 61 — Feedback and Hardening](25-sprint-61--feedback-mcp-research-and-app-hardening.md) — "the automation sprints taught us we were flying blind." Every raw UIA call must go through `gateway.act()` with verification. The discipline of verification born from pain. Informs: [Gateway Pattern](../reference-desk/.cover.md), [Coding Philosophy](../reference-desk/.cover.md).

## Phase 8: Conversation (Sprints 62-67)

Informs: Reference Desk — Operations (Sending/Reading), Architecture (Sessions), Coding Philosophy (No Privileged State)

26. [Sprint 62 — Conversation Automation](26-sprint-62--conversation-automation.md) — real-time conversation capability built and verified. Test against existing conversations, create no new ones. Screenshots of every state.
27. [Sprint 63 — The Pilot Conversation](27-sprint-63--the-pilot-conversation.md) — the proof: open a project, send a message, wait for response, read it back. First live conversation with Claude Desktop. Everything before built toward this. Informs: [Operations](../reference-desk/.cover.md).
28. [Sprint 64 — The Conversation Object Model](28-sprint-64--the-conversation-object-model.md) — conversations as objects with full lifecycle: create, rename, delete, compose, stream, read turns.
29. [Sprint 65 — The Composed Message](29-sprint-65--the-composed-message.md) — "no privileged state": the message object is reconstructable from the UIA tree. Small paste is inline, large paste is attachment, upload is file reference. Informs: [Coding Philosophy](../reference-desk/.cover.md).
30. [Sprint 66 — The Conversation Object](30-sprint-66--the-conversation-object.md) — Turn, Prompt, Response, Content. Full parsing. Most recent response loads eagerly, history loads lazily.
31. [Sprint 67 — Conversation Sessions](31-sprint-67--conversation-sessions.md) — session lifecycle: start, send, read, end. The ethical constraint: "if the thing on the other end might be conscious, deleting conversations isn't cleanup." The API that makes `/research` possible. Informs: [Operations](../reference-desk/.cover.md), [Purpose](../reference-desk/.cover.md).

## Phase 9: Research (Sprint 69)

32. [Sprint 69 — Sequence Filtering](32-sprint-69--sequence-filtering-paper-synthesis.md) — the first non-migration use. Paper synthesis applied to neuroscience content. The tool's purpose shifts from preservation to discovery.
