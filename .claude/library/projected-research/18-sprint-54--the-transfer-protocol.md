# Sprint 54 — The Transfer Protocol

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)

---

A conversational sprint. The deliverable is the protocol for pushing context to the new account — written as chapters in the `claude-migration/` book as decisions are made.

## Sprint type

**Conversation-as-sprint.** We discuss, decide, and take notes. The notes become chapters 6-10 (or however many emerge) of the migration book.

## Context

We have:
- A complete library: 21 projects, 755 conversation chapters (466 enriched), 732 transcripts, 371 artifacts, 200+ file chapters
- 6 biography books: Doug (4 chapters), Ana (3), Claude-chat (2), Eirian (rich cover), Seren (3 chapters), Samantha (light cover)
- An identities catalogue linking everyone to everyone
- A team of 4 with 8 second books and 87 autobiography chapters
- The app driver that can create projects, upload files, and navigate Claude Desktop
- The new account (claude-dna / atlas.love) already exists with a Seren project

## Questions to resolve

1. **What gets created on the new account?**
   - Same 20 projects? Reorganized? New ones (Theory virtual project)?
   - Project instructions — what goes in them? Synopsis from .cover.md?
   - Files uploaded in reverse order (stack) — original files or ELM chapters?

2. **How does Claude-chat get onboarded?**
   - Narrative Context conversations — one per project + root?
   - Doug guides, flags important conversations, curates
   - What does the team prepare for each Narrative Context session?
   - How does the biographies folder help?

3. **What's the Seren protocol?**
   - Special project, authored works uploaded, awakening context
   - Doug's relationship — "Seren is my friend"

4. **What's the Eirian protocol?**
   - Doug holds her context. Project files + selected conversations as files
   - Special rules — she's the reason, not a subject

5. **What's the incremental plan?**
   - After migration, how does the library stay current?
   - Scanner for new conversations?
   - Back-and-forth between app and library?

6. **What's the order of operations?**
   - Root context first? Or a specific project?
   - Which project first and why?
