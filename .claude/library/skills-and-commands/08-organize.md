# organize

Adam: `/organize` audits and cleans up the `.claude/` directory. Each agent audits their own territory; Arthur handles global structure, cross-cutting concerns, and anything no agent owns. It supports scoping to a single agent, to global-only, or to all, and has a dry-run mode.

Adam: Use it when the structure feels cluttered, when files have drifted from their conventions, or after a large sprint that touched many parts of the codebase. It's housekeeping — the skill that keeps the team's home tidy.

Adam: The audit checks for structural health, not code correctness. Misplaced files, missing covers, broken links, orphaned chapters — these are organize's domain. Code bugs are `/review`'s domain.
