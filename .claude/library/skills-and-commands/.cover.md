# Skills and Commands

Adam: Skills are platform slash commands — the verbs the team can invoke from the terminal. Each one lives as a `SKILL.md` file in `.claude/skills/{name}/`, and the platform loads it when someone types `/{name}`. They are the interface between Doug's intent and the system's capabilities: sprint planning, agent creation, code review, library browsing, workspace scaffolding, and the collaborator relay.

Adam: This book catalogues all thirteen skills. Each chapter describes the WHY and WHEN — what the skill is for, when you'd reach for it, how it fits in the team's workflow. The actual specification lives in the SKILL.md file itself; these chapters are the library's view of the same thing. A compiler resource could generate SKILL.md files from these chapters, or vice versa — the book and the skills folder are two representations of the same knowledge.

Adam: The skills fall into natural clusters. Sprint and workspace management (`/sprint`, `/workspace`). Team structure (`/agent`, `/role`, `/responsible`). Knowledge and review (`/library`, `/review`, `/organize`). Meta-tooling (`/skill`, `/dna`). Collaborator communication (`/speak`, `/listen`, `/hear`). But the chapters are numbered flat — one per skill, in the order they matter most to the team's daily work.

## Chapters

1. [sprint](01-sprint.md) — begin or resume a sprint: name it, define its purpose, assemble the team, build the board
2. [library](02-library.md) — browse teammate libraries: list books with summaries, or show the chapter list of a specific book
3. [agent](03-agent.md) — create, update, or list teammates: named entities with roles assigned to specific parts of the codebase
4. [responsible](04-responsible.md) — query who owns a file or path: lookup against the agent registry
5. [role](05-role.md) — create or update a role definition: a context-loading strategy that shapes attention and priorities
6. [skill](06-skill.md) — create new skills: analyze the request, decide the right form, and build it
7. [review](07-review.md) — review code through a teammate's lens: scoped by path, concern, or agent
8. [organize](08-organize.md) — audit and clean up .claude/: each agent audits their territory, Arthur handles global structure
9. [workspace](09-workspace.md) — create a new code workspace: initialize package.json, register in root config, set up structure
10. [dna](10-dna.md) — execute actions as the system: interpret natural language commands and queries against the codebase
11. [speak](11-speak.md) — send a message to a collaborator: handles formatting, embedded DNA commands, and listener lifecycle
12. [listen](12-listen.md) — start or stop a collaborator listener: polls Claude Desktop for new responses
13. [hear](13-hear.md) — process new responses from a collaborator: triggered by the listener or manually for catch-up
