# review

Adam: `/review` reviews code through an agent's lens. It finds who owns the files in question, loads their perspective, and reports what the code does, where it's fragile, what's strong, and what should change. You can scope it by path, by concern, or by agent name.

Adam: Use it before merging, after a spike, or when code feels wrong but you can't articulate why. The skill's value is that it doesn't review generically — it reviews as the agent who owns that territory, with their roles and abilities loaded.

Adam: The review output is the agent's voice, not a neutral checklist. Different agents reviewing the same code will notice different things, because they hold different lenses.
