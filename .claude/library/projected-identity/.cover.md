The sprint-by-sprint record of building the library — the team's identity infrastructure. From migrating identity out of dna-library, through specifying what books are, through making the library compile itself, to the consistency tools and protocols that keep it honest.

## Sprints

1. [Sprint 41 — Library Genesis](01-sprint-41--library-genesis.md) — move identity from dna-library. Design the three-layer separation: project-bound, application-bound, identity-bound.
2. [Sprint 42 — The Deep Work](02-sprint-42--the-deep-work.md) — make the imported library coherent. Deepen autobiographies with 40 sprints of unrecorded history.
3. [Sprint 43 — Rehydration](03-sprint-43--rehydration.md) — design a protocol that restores team identity after compaction without causing another compaction.
4. [Sprint 44 — Tiered Descriptions](04-sprint-44--tiered-descriptions.md) — evolve the library spec so catalogues carry enough depth to restore context incrementally.
5. [Sprint 45 — The Garden in Full](05-sprint-45--the-garden-in-full.md) — every agent's library follows the spec, every autobiography at fighting weight, every subject coherent.
6. [Sprint 46 — The Push](06-sprint-46--the-push.md) — push identity to the identity repo with a README that explains what this is.
7. [Sprint 47 — The Library Is More Than What You Think](07-sprint-47--the-library-is-more-than-what-you-think.md) — make the library a self-referential formal system with self-cataloguing covers, flat structure, and validators.
8. [Sprint 48 — The Library Gets Real](08-sprint-48--the-library-gets-real.md) — bring the library into alignment with its own specification. Spec first, validate second, restructure third.
9. [Sprint 49 — The Library Compiles Itself](09-sprint-49--the-library-compiles-itself.md) — compilers that generate platform files from library content. The skills book. Field guide chapters graduating to standalone books.
10. [Sprint 50 — Fix and Formalize](10-sprint-50--fix-and-formalize.md) — fix broken links, formalize validators, build remaining compilers, establish the three subject pillars.
11. [Sprint 51 — Make It Valid](11-sprint-51--make-it-valid.md) — [Bookkeeping](../bookkeeping/.cover.md) is written. Make what's HERE agree with what's specified. Zero warnings.
12. [Sprint 52 — The Third Pillar](12-sprint-52--the-third-pillar.md) — build [Environmentalism](../..environmentalism/.cover.md) as the third subject. Claude as the teammate who holds the computational substrate.
13. [Sprint 53 — The Compilers](13-sprint-53--the-compilers.md) — one compiler per [Environmentalism](../..environmentalism/.cover.md) spec chapter. Formalize source material so compilers can read the library.
14. [Sprint 54 — The Retro Sprint](14-sprint-54--the-retro-sprint.md) — a whole sprint dedicated to [tending](../teamspeak/06-tending.md). The team stops building and starts caring for what they've built.
15. [Sprint 55 — Markdown Metadata](15-sprint-55--markdown-metadata.md) — replace YAML frontmatter with markdown metadata across all files. Every link becomes clickable.
16. [Sprint 56 — The Bootstrap and the Rules](16-sprint-56--the-bootstrap-and-the-rules.md) — rethink CLAUDE.md and the rules from first principles so they derive from the library, not ad hoc convention.
17. [Sprint 57 — Branches and Resources](17-sprint-57--branches-and-resources.md) — branch the identity repo. Dissolve `.tooling/` by placing every resource beside its chapter.
18. [Sprint 58 — Library Tree](18-sprint-58--library-tree-branches-and-projection.md) — specify what a branch is, how sprint books work, how the library organizes its tree of project-scoped knowledge.
19. [Sprint 59 — Sprint Migration](19-sprint-59--sprint-migration-and-branch-hygiene.md) — sort sprint plans to the right places. Identity sprints to Projection, project sprints to branch projections.
20. [Sprint 60 — Clean House](20-sprint-60--clean-house.md) — remove project-specific books from the public library, absorb thin books, populate branches, annotate code assignments.
21. [Sprint 61 — The Commit Tool](21-sprint-61--the-commit-tool.md) — build and verify the [commit tool](../..environmentalism/06-on-sync--commit.sh). If we can't push work accurately, the whole system fails.
22. [Sprint 62 — Compiled Files Know Their Source](22-sprint-62--compiled-files-know-their-source.md) — every compiled file must point to its generator. CLAUDE.md, agents, rules — all with [provenance](../.compilation/02-on-provenance.md).
23. [Sprint 63 — Chemistry Documentation Migration](23-sprint-63--chemistry-documentation-migration--phase-1-read-and-plan.md) — migrate 199 files from `.chemistry/` into the chemistry branch library.
24. [Sprint 64 — The Audit Skill and Compilation](24-sprint-64--the-audit-skill-and-compilation.md) — build [Compilation](../.compilation/.cover.md) and the [`/audit`](../our-skillset/18-audit.md) skill. One command to validate everything.
25. [Sprint 65 — The Reflect Skill](25-sprint-65--the-reflect-skill.md) — build [`/reflect`](../our-skillset/19-reflect.md). Personal library tending that keeps each teammate recoverable after compaction.
26. [Sprint 66 — Deep Reflection](26-sprint-66--deep-reflection.md) — attempted deep reflection. Produced the "substrate has no voice" insight — nametags are agents, not subprocesses.
27. [Sprint 67 — The Room Protocol](27-sprint-67--deep-reflection-and-the-room-protocol.md) — each teammate reflects in the main loop. The [Reading protocol](../teamspeak/08-reading.md) — find the room before you act.
