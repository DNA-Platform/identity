How the library composes itself. Every compiler that generates a platform file, every validator that checks a constraint, every tool that routes changes to the right place — this catalogue inventories them all. Compilation is not just "the environment compiling outward." It is every automated process the library performs: validation is composition (does the structure hold?), sync is composition (do the branches agree?), audit is composition (does the whole thing cohere?).

[Environmentalism](../..environmentalism/.cover.md) specifies what each platform file must contain. Compilation catalogues the machinery that produces and verifies those files — and the machinery that validates the library's own internal structure. The distinction: Environmentalism says "CLAUDE.md must contain these sections." Compilation says "here is the compiler that reads the library and generates CLAUDE.md, here is the validator that checks the result, and here is the audit skill that runs them all."

Part of auditing the library is reading this catalogue carefully, because the audit IS the composition check. The `/audit` skill runs every process catalogued here. If a process is missing from this catalogue, the audit doesn't check it. If a process is catalogued here but not automated, that's a gap. Compilation is the specification of what "check everything" means.

## Specification chapters

1. [On Automation](01-on-automation.md) — what counts as an automated process in this library. A tool is automation if it reads library content and produces or validates an artifact. The test: could a human do this by hand? If yes and we automated it, it belongs here.
2. [On Provenance](02-on-provenance.md) — every generated file must link to its generator. The constraint, why it exists, how it's validated. Provenance is the thread that connects a compiled artifact back to the library source that produced it.

## Catalogue chapters

3. [Compilers](03-compilers.md) — every compiler. What it reads, what it generates, which [Environmentalism](../..environmentalism/.cover.md) chapter specifies it. Four compilers generate the platform: teammates, bootstrap, rules, skills.
4. [Validators](04-validators.md) — every validator. What it checks, what it catches, how it's orchestrated. Three validators check structure, links, and provenance. The runner pipes them in sequence.
5. [Tools](05-tools.md) — the commit tool, sync infrastructure, branch routing. Automation that isn't compilation or validation — the plumbing that moves content to the right place.

## The audit skill

The [`/audit` skill](../our-skillset/audit/.cover.md) runs every process catalogued here. It is itself an automated artifact with provenance — the self-referential loop: the audit checks provenance; the audit itself must satisfy provenance.
