# On Provenance

- **specification:** Provenance
- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)

---

Every generated file must link to its generator. This is the provenance constraint — the thread that connects a compiled artifact back to the library source that produced it.

## The rule

A file is "generated" if it was produced by a [compiler](03-compilers.md) rather than written by hand. Every generated file must contain a comment or metadata field that names the compiler that produced it and tells the reader not to edit the file directly.

The standard form is an HTML comment at the top of the file:

```
<!-- Generated by 01-on-teammates--compiler.ts. Edit the library, not this file. -->
```

The comment serves two audiences:
- **A teammate** who opens the file knows to edit the library source, not this output.
- **A validator** can check that the comment exists and that the named compiler is real.

## Why provenance matters

Without provenance, compiled files drift. A teammate edits CLAUDE.md directly, adding a section. The next compilation overwrites their edit. Or worse: no one recompiles, and the hand-edit persists as the only copy of that content — outside the library, outside version control's meaningful diff, outside the team's knowledge.

Provenance prevents this by making the relationship visible. The comment is not a formality — it is the mechanism that keeps compiled output honest.

## What the validator checks

The [compiled-links validator](../..environmentalism/07-on-compiled-links--validator.ts) checks that every file in `agents/`, `rules/`, and the root `CLAUDE.md` has valid provenance. Specifically:
- The file contains a comment naming its generator
- Every link in the file resolves to a real target

The `/audit` skill runs this validator as part of its full check. A file with broken provenance fails the audit.

## The self-referential case

The `/audit` skill is itself a generated file (produced by the skills compiler). It must satisfy the provenance constraint it checks. This is not a paradox — it's the same self-referential structure the library uses everywhere. The catalogue catalogues itself. The validator validates its own output format. The audit checks its own provenance.
