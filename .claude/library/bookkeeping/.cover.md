# Bookkeeping

Libby: The specification of what books are and how they work. This is an essential specification book for [Knowledge](../..librarianship/.cover.md) — it defines the terms that make the library possible. Each chapter is a specification chapter with a `specification:` label naming the term it defines.

## Specification chapters

1. [Anatomy of a book](01-anatomy-of-a-book.md) — `specification: Book`. Covers, frontmatter (`title > author > subject`), chapter signing, synopses, names as identity. Validator: [bookkeeping.ts](bookkeeping.ts)

## Chapters to write `[SCAFFOLD]`

2. Subject catalogue `[SCAFFOLD]` — `specification: Subject Catalogue`. The `.` prefix, self-cataloguing, `catalogues:` field, subject-shaped descriptions, multi-subject membership
3. Library catalogue `[SCAFFOLD]` — `specification: Library Catalogue`. The `..` prefix, scope rule, the public library exception for personal libraries
4. Chapter `[SCAFFOLD]` — `specification: Chapter`. Numbering, signing, sub-numbering
5. Specification `[SCAFFOLD]` — `specification: Specification`. What it means for a chapter or book to be a specification: the `specification:` label as a plain term (not a link), accompanying validator resources as chapter resources, how specifications evolve from chapters to books
