# Bookkeeping

Libby: The specification of how books work. Every book in the library is valid because this book defines what valid means. Each chapter specifies one term — a type, a practice, a structural element. Together they form the type system where dots mark specialization and self-reference constitutes identity.

## The dot type system

```
Chapter          ->  Cover (.cover.md)
                     ^ specialization: a chapter that knows about its book

Book (no prefix) ->  Subject catalogue (. prefix)  ->  Library catalogue (.. prefix)
                     ^ specialization:                  ^ specialization:
                     a book that catalogues              a subject catalogue
                     other books                         that IS the library
```

Libby: No prefix is the base type. `.` is one level of specialization. `..` is two. The dots are type markers, not sorting markers. A cover is a chapter that self-describes. A subject is a book that self-catalogues. A library is a subject that self-references.

## Specification chapters

1. [On Books](01-on-books.md) — `specification: Book`. The base type: a directory with a [cover](03-on-covers.md) and [chapters](02-on-chapters.md). What belongs in a book, what doesn't. The book as the library's atom — the smallest self-describing unit. [Resources](02-on-chapters.md#resources) sit beside chapters.

2. [On Chapters](02-on-chapters.md) — `specification: Chapter`. The base unit of content. Numbered files, [signed](05-on-frontmatter.md) by their author. Sections with heading anchors. The [resource pattern](02-on-chapters.md#resources): code beside the chapter that documents it.

3. [On Covers](03-on-covers.md) — `specification: Cover`. The specialized chapter: `.cover.md`, the book's face. Opening paragraph as [synopsis](09-on-synopsis.md). [Table of contents](03-on-covers.md#the-table-of-contents) with descriptions. How catalogue covers differ — they describe other books, not just chapters.

4. [On Names](04-on-names.md) — `specification: Name`. Tier-zero synopsis: the densest description, read in every link at zero cost. [Timeless](04-on-names.md#timelessness) naming. The [literary-to-literal spectrum](04-on-names.md#the-literary-and-the-literal): book names lean literary, specification terms lean precise.

5. [On Frontmatter](05-on-frontmatter.md) — `specification: Frontmatter`. The YAML metadata that connects files to the library. [Six fields](05-on-frontmatter.md#the-fields): title, catalogues, specification, author, coauthor, subject. [Labels vs links](05-on-frontmatter.md#labels-and-links). [Field order](05-on-frontmatter.md#field-order) by type.

6. [On Links](06-on-links.md) — `specification: Link`. Links ARE the hierarchy. [Discoverability](06-on-links.md#discoverability): a book no one can find doesn't exist — link generously, from many paths. Inline links, [section targets](06-on-links.md#section-targets), [citations](06-on-links.md#citations). The [direction convention](06-on-links.md#the-direction-convention).

7. [On Subjects](07-on-subjects.md) — `specification: Subject`. The `.` prefix: a book that catalogues other books. [Self-cataloguing](07-on-subjects.md#self-cataloguing). The `catalogues:` label. [Flat peers](07-on-subjects.md#flat-peers) — books beside, not inside. [Subject-shaped descriptions](07-on-subjects.md#subject-shaped-descriptions). [Multi-subject membership](07-on-subjects.md#multi-subject-membership).

8. [On Libraries](08-on-libraries.md) — `specification: Library`. The `..` prefix: a subject catalogue that IS the library. Catalogues subjects and [essential specifications](08-on-libraries.md#what-a-library-catalogue-catalogues). One per scope, with the [public library exception](08-on-libraries.md#one-per-scope). The library [knowing itself](08-on-libraries.md#the-library-knowing-itself).

9. [On Synopsis](09-on-synopsis.md) — `specification: Synopsis`. Four layers between question and answer: [name](09-on-synopsis.md#layer-0-the-name) -> [library catalogue](09-on-synopsis.md#layer-1-the-library-catalogue) -> [subject catalogue](09-on-synopsis.md#layer-2-the-subject-catalogue) -> [cover](09-on-synopsis.md#layer-3-the-book-cover) -> [chapter](09-on-synopsis.md#layer-4-the-chapter). [Budgets](09-on-synopsis.md#the-reading-cost-test) at each level. The [tending practice](09-on-synopsis.md#the-tending-practice).

10. [On Evolution](10-on-evolution.md) — `specification: Evolution`. How the library evolves: [chapter to book](10-on-evolution.md#chapter-to-book), [book to subject](10-on-evolution.md#book-to-subject), [catalogue evolution](10-on-evolution.md#catalogue-evolution), [dissolution](10-on-evolution.md#dissolution) of container labels. 11. [On Specifications](11-on-specifications.md) — `specification: Specification`. What makes a specification authoritative: the `specification:` label, [validators](11-on-specifications.md#validators) as executable contracts, [essential specifications](11-on-specifications.md#essential-specifications) per subject. ## The validator

Libby: [bookkeeping.ts](bookkeeping.ts) checks what these chapters specify: frontmatter fields and order, chapter signing, cover structure, catalogue conventions. One validator for the whole book — the specifications form a coherent type system, not independent rules. Run it with `npx tsx bookkeeping.ts <library-path>`.
