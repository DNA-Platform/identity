# Bookkeeping

Libby: The specification of how books work. This book defines the types that make the library possible and the relationships between them. Each chapter specifies one type. Together they form a type system where dots mark specialization.

Libby: The type hierarchy:

```
Chapter          →  Cover (.cover.md)
                     ↑ specialization: a chapter that knows about its book

Book (no prefix) →  Subject catalogue (. prefix)  →  Library catalogue (.. prefix)
                     ↑ specialization:                ↑ specialization:
                     a book that catalogues             a subject catalogue
                     other books                        that IS the library
```

Libby: No prefix is the base type. `.` is one level of specialization. `..` is two. The dots aren't sorting markers — they're type markers. A cover is a chapter that self-describes. A subject is a book that self-catalogues. A library is a subject that self-references.

## Specification chapters

1. [The anatomy of a book](01-anatomy-of-a-book.md) — `specification: Book`. The base type: a directory with a cover and chapters. Frontmatter fields. Chapter signing. Names as synopsis. Validator: [bookkeeping.ts](bookkeeping.ts)

2. Cover `[SCAFFOLD]` — `specification: Cover`. A specialized chapter: `.cover.md`, the entry point to a book. Carries `subject:`, optionally `catalogues:` and `specification:`. The cover IS the book's face.

3. Subject catalogue `[SCAFFOLD]` — `specification: Subject Catalogue`. A specialized book: `.` prefix, self-cataloguing, `catalogues:` label, subject-shaped descriptions. Books sit beside it as peers.

4. Library catalogue `[SCAFFOLD]` — `specification: Library Catalogue`. The most specialized book: `..` prefix, catalogues subjects and essential teammates. One per scope (with the public library exception for personal libraries).

5. Chapter `[SCAFFOLD]` — `specification: Chapter`. The base unit: numbered file with `title:` and `author:`. Sub-numbering for complex chapters. Sections with heading anchors.

6. Specification `[SCAFFOLD]` — `specification: Specification`. A label on a chapter or book: `specification: {term}`. The term is a word, not a link. Accompanied by a validator resource (a `.ts` file sharing the chapter's name). How specifications evolve from chapters to books.

## The dot type system

Libby: The dots encode inheritance:
- **(no prefix)** — base type. Books and chapters. The things that hold content.
- **`.` prefix** — one level of specialization. Subject catalogues. Covers (`.cover.md`). The things that organize and describe.
- **`..` prefix** — two levels of specialization. Library catalogues. The things that ARE the library.

Libby: A cover IS a chapter (same `.md` file format) but specialized with extra fields. A subject catalogue IS a book (same directory format) but specialized with self-cataloguing. A library catalogue IS a subject catalogue but specialized to the library level. Each level adds behavior while preserving the base structure.
