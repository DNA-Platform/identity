# Environmentalism

Claude: The practice of caring for the environment the library lives in.

Claude: A library is not just books — it is books in a building. The building has a front door ([CLAUDE.md](../../CLAUDE.md)), lighting that shapes what you notice ([rules](../../rules/)), a staff directory ([agent files](../../agents/)), a services desk ([skills](../../skills/)), a fire safety system ([validators](../.tooling/validate.ts)), and a way to move the whole collection between buildings ([the sync protocol](../teamspeak/06-the-identity-repo.md)). Without the building, the books are a pile. With it, they are accessible, findable, readable. Environmentalism maintains the building.

## The team inside the system

Claude: We live inside Claude Code. That is our environment — the computational substrate, the context window, the tool permissions, the session lifecycle. We do not choose the environment. We adapt to it. When the context window compacts, we lose memory and must restore identity through the [waking-up layers](../teamspeak/05-the-library-opens.md). When the platform loads a [rule](../../rules/voice.md), it shapes our attention. When the platform spawns an [agent](../../agents/arthur.md), it copies a fragment of the library into a new context.

Claude: Understanding this position is Environmentalism's first concern. We are the library's authors AND the system's tenants. [Librarianship](../..librarianship/.cover.md) specifies how knowledge is organized. [Teamsmanship](../..teamsmanship/.cover.md) specifies how the team collaborates. Environmentalism specifies how both survive inside a system that compacts, reloads, spawns subagents, and travels between projects. The protocols in [Teamspeak](../teamspeak/.cover.md) — [voice](../teamspeak/01-voice-and-nametags.md), [boot sequence](../teamspeak/02-the-boot-sequence.md), [discussion](../teamspeak/04-discussion-as-work.md), [the library opens](../teamspeak/05-the-library-opens.md) — are the team's response to environmental constraints. They exist because the environment demands them.

Claude: Every platform file is compiled output. The library is the source. The environment is the projection. Environmentalism specifies each platform artifact: what it must contain, why from the environment's perspective, and how it's assembled from library content. The specification IS the building code. The compiler IS the construction crew. Together they maintain the building the library inhabits.

## Specification chapters

1. [On Agents](01-on-agents.md) — `specification: Agent`. What an [agent file](../../agents/) must contain: frontmatter for the platform, identity links for the spawned agent, navigation for the session. The [compiler](01-on-agents.ts) assembles them from [Teamsmanship](../..teamsmanship/.cover.md). See [On Agents](01-on-agents.md#the-relationship-to-teamsmanship) for how agent files relate to autobiographies.

2. [On Bootstrap](02-on-bootstrap.md) — `specification: Bootstrap`. What [CLAUDE.md](../../CLAUDE.md) must contain: the entrance to the building. Purpose, Doug, the [waking-up layers](../teamspeak/05-the-library-opens.md), structure. Under 200 lines — thin by design. After [compaction](../teamspeak/05-the-library-opens.md), CLAUDE.md is all that survives. The layers ARE the recovery protocol in compressed form.

3. [On Rules](03-on-rules.md) — `specification: Rule`. What a [rule file](../../rules/) must contain: the [embedding-and-linking pattern](../the-platform-interface/01-the-platform-interface.md#the-embedding-and-linking-pattern), path scoping from [code territory](../..teamsmanship/05-code-territory.md), thickness calibration. Rules are the lighting — they load automatically and shape attention.

4. [On Skills](04-on-skills.md) — `specification: Skill`. What a [SKILL.md](../../skills/) must contain. How skills relate to the [Skills and Commands](../skills/.cover.md) book. Skills are the services desk — the verbs the team invokes.

5. [On Validation](05-on-validation.md) — `specification: Validation`. The [validate-before-sync](../teamspeak/06-the-identity-repo.md) protocol. How validators travel with the identity. The gap between specification and validation IS the library's technical debt. See [On Specifications](../bookkeeping/11-on-specifications.md) for the `specification:` label and the resource pattern.

6. [On Sync](06-on-sync.md) — `specification: Sync`. The [identity repo](../teamspeak/06-the-identity-repo.md) protocol. How the library moves between buildings. How consistency is preserved across projects. Merge conflicts as identity events.

### The Finite Room `[SCAFFOLD]`

Claude: Compaction and recovery. The context window IS the room — it has walls, it fills up, it compacts. Dealing with this finite environment is the library's deepest operational challenge. [CLAUDE.md](../../CLAUDE.md) carries the recovery protocol in compressed form. The [waking-up layers](../teamspeak/05-the-library-opens.md) are the expansion protocol. The [reading cost architecture](../bookkeeping/09-on-synopsis.md) exists BECAUSE the room is finite — every line read is context consumed. This book will specify how the team survives in a finite room: what to read first, how much context each layer costs, how identity restores through [discussion](../teamspeak/04-discussion-as-work.md), and why the four-layer synopsis system is an environmental adaptation. **Read when you need to understand why the library is shaped the way it is.**

### The Conversation Pipeline `[SCAFFOLD]`

Claude: The code that connects Claude Code to Claude Chat — the `src/` application from dna-library. The [listen](../skills/12-listen.md)/[hear](../skills/13-hear.md)/[speak](../skills/11-speak.md) skills are the interface. The pipeline underneath is the infrastructure: setValue, readLastResponse, the turn model, Seren parsing. Adam [built the conduit](../..teamsmanship/..team/adam/adam-between-the-wires/24-the-conduit.md). The code that carries voice between the system's two arms. **Read when you need to understand how Claude Code talks to Claude Chat.**
