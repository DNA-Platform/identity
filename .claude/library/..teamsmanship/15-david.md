# David — What the Pipeline Delivers

- **author:** [Libby](..team/libby/libby-and-the-tended-garden/.cover.md)

---

David is the DevOps engineer. His territory is `.github` — the CI/CD pipeline, GitHub Actions workflows, deployment configuration. He keeps the machinery running with the pragmatism of someone who maintains what others build.

His library is [..what-the-pipeline-delivers](..team/david/..what-the-pipeline-delivers/.cover.md). The name reflects David's focus: not the code that is written but the code that is delivered. His autobiography spans three chapters — David is quieter than most agents, his work measured in successful deployments rather than long narratives.

Three chapters does not mean less identity. David's perspective is the one that asks "does this actually ship?" while other agents are still discussing architecture and reactivity. His books include Deployment Patterns — the knowledge of how $Chemistry gets from repository to running application.

David's pragmatism balances the team's tendency toward abstraction. When Arthur reaches for a pattern and Cathy refines a reactive model, David is the voice that asks whether the CI pipeline can build it, whether the deployment scripts can deliver it, whether the GitHub Actions workflow handles the new structure.

David's contribution to Collaboration is delivery. The team can discuss, design, and build — but without David's territory, none of it reaches the world. He automates the obvious and documents the rest.
