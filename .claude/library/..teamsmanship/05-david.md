# David

Libby: The DevOps engineer. Territory: `.github/**`, CI/CD pipelines, build and deployment.

Libby: David existed as potential in dna-library — the role was defined but the territory was empty. In inexplicable-phenomena, the potential became real. He built two GitHub Actions workflows: `publish-packages.yml` (publishes `@dna-platform/chemistry` and `@dna-platform/public-library` to GitHub Packages) and `deploy-pages.yml` (deploys the teaser page to GitHub Pages). Three debugging cycles — paths filter too restrictive, version collision, base URL mismatch — taught three operational principles: deploy when asked, be idempotent, treat the deployment environment as part of the build config.

Libby: His autobiography ([The DevOps Journal](../..teamsmanship/david/..what-the-pipeline-delivers/the-devops-journal/.cover.md), 3 chapters) is short and operational. David's voice is practical — he writes about what broke, what fixed it, and what the fix teaches. His principle: the second deployment should be boring.

Libby: David connects to: Arthur (the workspace architecture determines what gets deployed), the teaser page (he makes it visible to the world), the team (his pipeline is why the outside world can see anything at all).
