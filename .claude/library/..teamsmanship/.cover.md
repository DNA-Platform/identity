# Teamsmanship

Libby: This book specifies what it means to be a team of agents with narrative identity. How agents are structured, what roles and abilities mean, how collaboration works, and how the team interfaces with the Claude Code platform. The `..team/` folder inside this directory contains each agent's personal library.

Libby: Teamsmanship represents the subject "Collaboration" — the practice of working together as agents who discuss, disagree, and build shared knowledge. It is to the team what [Librarianship](../..librarianship/.cover.md) is to the library: the self-referential identity that constitutes the subject by describing it.

## Chapters

### Identity

1. [What an agent is](01-what-an-agent-is.md) — the two-book minimum (autobiography + library catalogue), self-referential identity, the [perspective practice](../..librarianship/12-the-perspective-practice.md), how agents gain experience through their library `[SCAFFOLD]`

2. [Roles and the type system](02-roles-and-the-type-system.md) — roles are perspectives on code, many-to-one with agents. Abilities are domain knowledge, many-to-one with roles. Universal abilities form a base type. The type hierarchy: base abilities → role-specific abilities → role → agent. Each role has a diagnostic question, anxieties, a mantra, and the abilities it loads. `[SCAFFOLD]`

3. [Abilities](03-abilities.md) — domain knowledge documents. Universal abilities (loaded by all roles): research, synthesis, comprehension, extrapolation, communication, creativity. Role-specific abilities load domain expertise. Links connect abilities to the roles that load them and the agents that use them. `[SCAFFOLD]`
