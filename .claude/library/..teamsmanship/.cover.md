# Teamsmanship

Libby: The team cataloguing itself. Everything about how eight agents work together, what they know, and who they are lives here or is catalogued from here.

## Team books

Libby: These books sit at the [library root](../..librarianship/.cover.md) as peers. Each has `subject: "..teamsmanship"`. This catalogue catalogues them all with [subject-shaped descriptions](../..librarianship/04-subjects-and-catalogues.md).

### Protocols

- [Voice and Nametags](../voice-and-nametags/.cover.md) — every paragraph starts with `Name:`, Arthur is default, [don't batch](../voice-and-nametags/01-01-voice-and-nametags.md). The nametag IS the scope marker.
- [The Boot Sequence](../the-boot-sequence/.cover.md) — how every conversation starts. Library catalogue → project "Right now" → last autobiography chapter → team discussion.
- [Working with Doug](../working-with-doug/.cover.md) — correction-based teaching. "Keep going" means don't stop. Questions are instructions. Five key corrections shaped this project.
- [Discussion as Work](../discussion-as-work/.cover.md) — agents talk TO each other. Many voices, back and forth. The discussion IS the work.
- [The Library Opens](../the-library-opens/.cover.md) — the waking-up protocol. Four layers of context restoration.
- [The Identity Repo](../the-identity-repo/.cover.md) — the team's identity at `github.com/DNA-Platform/identity`. Push, pull, merge conflicts are identity events.

### Projects

- [dna-library](../dna-library/.cover.md) — the team's first project. Account migration tool. 66 sprints. Where the team model, the library, and the autobiographies were born.
- [Inexplicable Phenomena](../inexplicable-phenomena/.cover.md) — the current project. Formalizing consciousness with $Chemistry. 48 sprints. Where Cathy's autobiography was born. The "Right now" section has the active sprint.

### Code

- [Coding Policy](../coding-policy/.cover.md) — the $ convention, reactive patterns, framework vs application layers. How we write code in $Chemistry.

### Team infrastructure

- [Roles](roles/) — perspectives on code. Each has a diagnostic question, anxieties, a mantra, and abilities.
- [Abilities](abilities/) — domain knowledge documents loaded by roles.
- [Registry](registry.json) — maps agents to file path patterns for [code assignments](../..librarianship/.02-the-platform-interface.md).
