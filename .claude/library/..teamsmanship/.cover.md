# Teamsmanship

Libby: This book specifies and catalogues the subject of Collaboration — what it means to be a team of teammates with narrative identity. The specification chapters describe how agents work. The catalogue chapters describe each book and each teammate library that belongs to this subject.

Libby: The `..team/` folder inside this directory contains each teammate's personal library — whole libraries living inside a subject catalogue because they ARE Collaboration. This is the [exception](../..librarianship/11-the-flat-structure.md) to the flat rule, documented in [chapter 01](01-what-an-agent-is.md).

## Specifying Collaboration

1. [What a teammate is](01-what-an-agent-is.md) — the two-book minimum (autobiography + library catalogue), self-referential identity, the [perspective practice](../..librarianship/12-the-perspective-practice.md), how agents gain experience through their library `[SCAFFOLD]`

2. [Roles and the type system](02-roles-and-the-type-system.md) — roles are perspectives on code (many-to-one with agents), abilities are domain knowledge (many-to-one with roles), universal abilities form a base type. Ten roles defined with inheritance, diagnostic questions, and ability links.

3. [Abilities](03-abilities.md) — the domain knowledge documents. Universal abilities as base type. Role-specific abilities. Links connecting abilities to roles to agents. `[SCAFFOLD]`

4. [Protocols](04-protocols.md) — the six operating conventions in the [protocols book](../protocols/.cover.md): voice, boot sequence, Doug, discussion, waking up, identity repo

5. [Code territory](05-code-territory.md) — who owns what paths, how territory connects to roles, how [rules](../../rules/) wire knowledge to code `[SCAFFOLD]`

6. [The agents folder](06-the-agents-folder.md) — how [`.claude/agents/`](../../agents/) interoperates with this book `[SCAFFOLD]`
