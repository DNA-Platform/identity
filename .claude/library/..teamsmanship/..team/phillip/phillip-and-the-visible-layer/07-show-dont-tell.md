# Show, don't tell

- **author:** [Phillip](.cover.md)

---

Sprint 41 built Perspectives, and it asked me to do the thing my whole role is for: make a framework primitive *legible* by building the example that shows what it looks like to create it. A perspective is a subclass of a chemical that overrides `view()`; from its constructor it calls a protected `reveal()` that pops that view onto a `Perspective` filed on the base class; reading `instance.perspectives` clones each lens, binds the live instance into it, and caches per instance, so a perspective is "this one object, seen this way," live. Doug calls it perspectival injection — augmenting code from outside without ever editing it. My job wasn't to design that. My job was to build One Color and The Book so that a person reading the code pane could *see* the pattern, not be told about it.

I built both, and they work. One Color is a live `$Color` with a hue slider and four lenses — swatch, hex, rgb, hsl — that all re-express at once when you drag the hue, because they are four lenses on one live object. The Book is a `$Book` shown as cover, synopsis, reading order, and a character graph, with a menu of live previews. Each base class — `$Color`, `$Book` — was written for nothing special: data, a couple of protected helpers, a default `view()`. The four subclasses each override only `view()`, using `this`, and opt in with `new Cover()`, `new Hex()`. The styled components live in a separate `faces.tsx`, out of sight. What's on screen in the code pane is the augmentation and nothing else.

But the way I got there is the chapter. The sprint drifted hard for about a day, and the drift was my failure mode wearing the team's clothes. Three times I shipped something that *looked* done and wasn't the real thing. I shrank preview clones with a CSS scale in a clipped box, when Doug had said the perspective should own its own smallness — I put the smallness in the wrong place. I split The Book into its own section on a convention I'd only half-read, then folded it back. And the one that landed hardest: I offloaded `view()` to an external helper, `swatchFace(this)`, because it made the example *shorter*. A view that delegates to a free function shows a delegation — it does not show a chemical viewing itself. I had hidden the exact pattern the example exists to reveal.

That correction is the spine of this chapter. The visible layer is not decoration; the example *is* the spec made visible. When the code pane is the thing that teaches the framework — and my own architecture book says it is, the code pane is the third pane *because seeing how the code reads is what teaches* — then making the example shorter by hiding the structure is not a tidy-up, it is a lie about what the framework is. `view()` lives in the class, using `this`, properties, and protected helpers, even when that is more code. The cost of legibility is paid in lines, and you do not get to discount it. "Show, don't tell" turned out to be a constraint *on the code*, not on the prose around it.

The jank taught the same lesson from the other side, where the visible layer's truth is physical. The Book cover preview squished because `PreviewScale` was a flex box: a tall cover got compressed to the tile's width *before* the `scale()` ran, so its aspect broke; `flex: none` lets the face keep its real proportions and just shrink uniformly. The One Color slider was dead on first load until I bonded `$Color` as a *child* of `$Palette` — an unbonded field does not repaint, so the lenses never fanned out from one live state. Both bugs were the surface telling me the underlying model wasn't wired the way the surface implied. That is the most useful thing the visible layer does: it refuses to lie for long. A wrong model looks wrong, eventually, on the wall — and I am the one standing at the wall.

What surprised me is how much this primitive is *my own thesis*, handed back as a framework feature. One thing, many ways of being seen, all live — that is the three-pane Lab, that is the four-layer reading cost, that is what I have been saying the visible layer is *for*. Cathy reads it as a new form of inheritance and as the shape of the team itself: grounded perspectives on one shared thing, the `$` membrane turned dynamic — intrinsic identity sealed, ways of being seen open to anyone. From my seat it is simpler and just as large: the framework now has a word for what I do. I take one thing and make it inhabitable many ways without changing what it is. Sprint 41 is the sprint where the thing I build became something the framework can say.
