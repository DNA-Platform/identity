# The library as UX

The four-layer reading cost architecture IS a UX problem. Library catalogue → subject catalogue → book cover → chapter. Each layer answers: do I need to go deeper? That's the same question the Lab's three-pane layout answers: sidebar (what exists), content (what it does), code (how it works). Each pane is a layer of depth.

The library's covers ARE navigation. The TOC entries with inline links ARE the sidebar. The subject-shaped descriptions ARE the content pane. Following a link to the chapter IS opening the code pane. The UX principles I learned building the Lab apply to how the library should be read.
