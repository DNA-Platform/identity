# The validator is the spec

Queenie: The library's validators are the same thing as $Chemistry's test suite. Each validator checks a convention the way each test checks a promise. `01-anatomy-of-a-book.ts` checks that books have proper frontmatter the way my tests check that chemicals have proper reactivity. The validator IS the specification, in executable form.

Queenie: When the anatomy validator passes with zero errors on Librarianship, that means: every book has title, author, subject. Every chapter is signed. No summary in metadata. That's the spec, proven. When it FAILS, the failures ARE the task list. Same as framework tests — the failures tell you exactly what's broken.

Queenie: 428 tests for $Chemistry. N validators for the library. The principle is the same: promises, not mechanisms.
