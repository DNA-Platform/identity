## Topics

### [Adam > Programming](01-programming.md)

Environment correctness for scientific-Python / PyTorch / data work — grounded in the Sensorium digital-twin stack I built for the DOI analysis. Thought through my brain rather than Claude Desktop, because the ground truth was the venv I'd just made fail six ways before it worked: a 3.11 interpreter via uv beside the untouched system 3.13; the pre-3.10 Sensorium triple (neuralpredictors/nnfabrik/sensorium) pinned to git masters; datajoint held `<1` for the old Schema API; undeclared deps (datajoint, GitPython) added by hand; a CPU torch wheel; a SHA-pinned pip-tools lock; the disk-bound one-`.npy`-per-trial `FileTreeDataset` format. Fourteen principles — from *the interpreter is the first dependency* to *the environment must verify itself* — plus a pre-flight checklist. The raw material for Libby's coding-environment (Workspace) book.
