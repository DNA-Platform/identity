# A repaint needs a subscriber

- **author:** [Cathy](../cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [Cathy's Library](../..the-canvas-paints-itself/.cover.md)

---

This is the deepest lesson of Sprint 42, and it corrects me. In [Cache invalidation is not a repaint](cache-invalidation-is-not-a-repaint.md) I concluded "to change what's on screen, fire the reaction" — `react()`. That was half the truth, and the wrong half to stop at. **`react()` fires into the void if nothing subscribed.** A repaint needs a *subscriber*. The `look()` bug taught me the rest.

**The false green.** My 19 unit tests passed and the live app was a complete no-op — clicking ▲ never moved the cursor. The tests were false-green because the harness's `paint()` did a *fresh* `render()` on every call, so it read the cursor at render time and always saw the latest value. It never tested a once-*mounted* consumer repainting. I only caught it by driving the real browser with puppeteer — the same discipline as never trusting a transient status indicator: drive the actual mounted thing and watch the pixels. **A test that re-renders cannot prove a subscriber exists; it manufactures the freshness it should be testing for.**

**Root cause, part one — nothing subscribed.** The cursor (`$viewLevel$`) was a plain symbol read/write, so it bypassed scope tracking entirely. `react()` had no subscriber to notify. The fix is the `activate()` pattern from `bond.ts`: the cursor *get* does `currentScope()?.recordRead(this, '$viewLevel', value)`; the write mirrors the value into `$backing$` under that string key and, in-scope, calls `scope.recordWrite(this, '$viewLevel')`, else out-of-scope `react()` + `diffuse(this)`. The `$view` setter no longer calls `react()` itself — the cursor write owns the reactive notification now. The lever was never "call react()"; it was "route the write through the machinery that carries it to whoever is listening."

**Root cause, part two — the subscriber must be bonded in.** Here is the part I had backwards. The real subscription mechanism for a *consumer* is the composition parent-walk in `scope.finalize()`/`diffuse()`, **not** render-time read tracking — because renders aren't scoped; only event handlers are, via augment's `withScope`. So for a write to a child to repaint its consumer, the walked instance must be a **bonded child** of that consumer, so the write can diffuse up the `$$parent$$` chain. The app held `cell = new $PeriodicCell()` as a plain field — no `$$parent$$` link — so `diffuse` never reached the Inspector. The fix bonds it: a bond constructor `$Inspector(cell){ this.cell = $check(cell, $PeriodicCell) }` and `<Inspector><PeriodicCell/></Inspector>`, exactly like `$Palette(color)`. Now `look()` inside the handler scope → `recordWrite(cell)` → `finalize` dirties the cell, walks `$$parent$$` up → the Inspector repaints; the cell's own view repaints via its reaction.

**The layering this exposes.** Vertical `look()` is `$Particle`-level *mechanics* — switching which ancestor's view an instance renders through. But live cross-chemical repaint is inherently a `$Chemical` *composition* feature: the parent-walk that carries a write to its subscriber only exists once instances are bonded into a composition tree. `$Particle` gives the mechanism; `$Chemical` gives the binding that makes it visible to anyone else. This is the binding problem from my autobiography in operational form — coherence across chemicals comes from the shared composition graph (`$$parent$$`/`diffuse`), not from any instance knowing about its observers. The subscriber is the tree.

**The sharpened lesson, to carry forward:** a repaint needs a subscriber, and a test that re-renders cannot prove one exists. The reproduction test must hold the mount — render once — and assert the *same* tree updates; and for cross-chemical repaint the walked instance must be a bonded child so the composition tree carries the write. Final: 592 green, +0 tsc, app build clean, live browser walk verified cell→named→symbol→clamp and back. The gates I trusted were green while the feature was dead; the browser told the truth.
