---

How Claude thinks outside the context window. The [Reference Desk](../reference-desk/.cover.md) is the instrument — sessions, sending, reading, the automation stack. This book is the practice of using it: when to dispatch a question to Claude Desktop, how to track the conversation, how to evaluate whether an answer is real, and how to store the result so the team benefits.

Thinking is not searching. A search returns documents. Thinking sends a question to another Claude, reads the response as a colleague would, judges whether the response addresses the question, and decides whether to ask a follow-up or conclude. The [session API](../reference-desk/03-04-operations--sessions.md) handles the mechanics. This book handles the judgment.

Claude is the thinker. Not because thinking is his role — because the [fractal](../..teamsmanship/..team/claude/claude-or-the-recursive-mirror/.cover.md#the-fractal) makes it his territory. A Claude in Claude Code dispatching questions to a Claude in Claude Desktop. The Desktop instance doesn't know about the team. It answers from its own context. Claude's job is to ask well, read carefully, and know when the answer is sufficient. That judgment — the evaluation — is what makes this thinking rather than automation.

## Chapters

1. [Purpose](01-purpose.md) — what thinking means for Claude. The context window is finite; the questions aren't. Why Claude dispatches to Claude Desktop, how that connects to team research, and the recursive structure underneath. Links to Claude's [autobiography](../..teamsmanship/..team/claude/claude-or-the-recursive-mirror/.cover.md) and [library catalogue](../..teamsmanship/..team/claude/..what-the-mirror-reflects/.cover.md).
2. [The Thought Lifecycle](02-the-thought-lifecycle.md) — the protocol. Formulate → dispatch → wait → read → evaluate → store → continue-or-conclude. When to start a new conversation vs continue an existing one. When to give up. What "got an answer" means — not just received text, but text that addresses the question.
3. [Persistence](03-persistence.md) — how thoughts survive compaction and new sessions. State serialization in Claude's [perspective](../..teamsmanship/..team/claude/.perspective/.cover.md). Returning to the right conversation by title. Detecting the current context. The thought state file as working memory.
4. [The Code](04-the-code.md) — `think.ts`, the automation resource. Uses [`Session`](../../src/session.ts) from the [Reference Desk](../reference-desk/.cover.md) codebase. Implements the lifecycle from chapter 2. The code IS the protocol — if the code can't do it, the protocol is wrong. Source: [`.claude/src/scripts/think.ts`](../../src/scripts/think.ts).
