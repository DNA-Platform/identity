**This is the book that catalogues the `.claude/src/` codebase.** If you are writing, modifying, or debugging any automation script — read this book first. Run the [introspect tool](09-codebase-index--introspect.ts) to see what methods already exist. Do not write code without reading what's here.

The Claude Desktop automation tool — how the team reaches outside the context window to research, converse, and discover. The code lives at `.claude/src/`. The tool mirrors the app: [`Claude`](../../src/claude.ts) → Pages → Components → Controllers → Infrastructure. The [`Session`](../../src/session.ts) class manages conversation lifecycles. Scripts read like English descriptions of what a human would do. If they don't, the abstraction is wrong — change the code, not the script.

This book is a living guide to an evolving codebase. It describes architecture as it IS, notes where the code is going, and links every claim to the source file that implements it. The [Projected Research](../projected-research/.cover.md) tells the story of building it. [Thoughtfulness](../thoughtfulness/.cover.md) describes the practice of using it — how Claude dispatches questions, evaluates responses, and stores results. The [/think skill](../our-skillset/20-think.md) uses [`Session.send()`](03-04-operations--sessions.md) for all Desktop interaction. This book tells you how to use the tool and how to extend it.

## Chapters

1. [Purpose](01-purpose.md) — what the tool is, why it exists, the ethical constraint: conversations are not disposable.
2. The Architecture (part) — the five-layer stack. Source: [`.claude/src/claude.ts`](../../src/claude.ts).
   - 02-01 [Layers](02-01-the-architecture--layers.md) — Scripts → [`Claude`](../../src/claude.ts) → Pages/Components → Controllers → Infrastructure. Controllers split into [sensors and actuators](02-02-the-architecture--gateway.md#sensors-and-actuators). Run the [introspect tool](09-codebase-index--introspect.ts) on any source file to see its API.
   - 02-02 [The Gateway Pattern](02-02-the-architecture--gateway.md) — act once, look repeatedly. `waitFor` (poll a sensor), `read` (poll until valid), `act` (action once + verify). The [coding philosophy](05-coding-philosophy.md) elevator metaphor: open your eyes and look, don't close them and wait.
   - 02-03 [Navigation](02-03-the-architecture--navigation.md) — screen detection in [`navigator.ts`](../../src/navigator.ts). `detectScreen()`, `requireScreen()`, `goHome()`. Also: `newChat()`, `checkConversation(id)`, `openConversationById(id)`, `dismissDialogs()` on the [`Claude`](../../src/claude.ts) class. [Lazy rendering](02-04-the-architecture--app-model.md#lazy-rendering): UIA reads what's visible, not what exists — scroll to materialize.
   - 02-04 [The App Model](02-04-the-architecture--app-model.md) — active page, lazy data, idempotent scripts, [lazy rendering](02-04-the-architecture--app-model.md#lazy-rendering). All foreground methods are idempotent — `maximize()`, `focus()` check state before acting.
3. Operations (part) — what scripts can DO.
   - 03-01 [Sending Messages](03-01-operations--sending.md) — `compose()` (clears draft first), `send()` (waits for response), `sendAndForget()` (confirms streaming started, doesn't wait), `sendAsync()` (returns immediately), `say()` (one-liner). Paste vs type: paste for large text, type for inline. Source: [`composer.ts`](../../src/components/composer.ts).
   - 03-02 [Reading Responses](03-02-operations--reading.md) — `checkStreaming()`, `readTurns()`, `readLastResponse()`, `scrollToBottom()`, `isResponseComplete()`. Source: [`conversation.ts`](../../src/pages/conversation.ts).
   - 03-03 [Project Operations](03-03-operations--projects.md) — open, create, files, instructions, [adding conversations to projects](03-03-operations--projects.md#adding-a-conversation-to-a-project) via the object chain: `ChatItem → ChatMenu → ProjectPicker`. Source: [`project.ts`](../../src/pages/project.ts), [`chat-list.ts`](../../src/components/chat-list.ts).
   - 03-04 [Sessions](03-04-operations--sessions.md) — `startSession()`, `session.send()`, `session.end()`. **The right tool for [/think](../our-skillset/20-think.md)** — handles the full conversation lifecycle: foreground, compose, send, wait, read, minimize. Source: [`session.ts`](../../src/session.ts).
4. Platform (part) — the OS interface layer. Windows-only.
   - 04-01 [Windows UIA](04-01-platform--uia.md) — `--force-renderer-accessibility`, the accessibility tree. Source: [`uia.ts`](../../src/uia.ts).
   - 04-02 [Win32](04-02-platform--win32.md) — window management, launch, focus. Source: [`window.ts`](../../src/window.ts).
   - 04-03 [The Shell](04-03-platform--shell.md) — persistent PowerShell, 12ms per call. Source: [`shell.ts`](../../src/shell.ts).
5. [Coding Philosophy](05-coding-philosophy.md) — the app reports its own readiness. If you work around the abstraction, change the code. Target semantics not presentation. No fixed waits — always test. The elevator metaphor: open your eyes and look.
6. [Writing Scripts](06-writing-scripts.md) — patterns, recipes. Scripts should be THIN — use Session, use the Claude class, don't duplicate.

9. [Codebase Index](09-codebase-index.md) — the [introspect tool](09-codebase-index--introspect.ts). Reads `.ts` files and outputs their public API — class names, method signatures, exports. **Run this before writing any automation code.**
10. [Architecture Patterns](10-architecture-patterns.md) — **READ THIS BEFORE WRITING CODE.** The MVC pattern: Controllers have [sensors and actuators](02-02-the-architecture--gateway.md#sensors-and-actuators), Views build typed objects with state verification, the object graph mirrors the app. Every action gets a confirmation read. The [ChatList chain](03-03-operations--projects.md#adding-a-conversation-to-a-project) is the reference implementation.
11. [Code-Library Linkage](11-code-library-linkage.md) — how code and library stay connected. The `///:` annotation convention: markdown with links, written in source files, read by the [introspect tool](09-codebase-index--introspect.ts) and validated by the [link checker](../..environmentalism/05-on-validation--check-links.ts). Code references the library. The library describes patterns. The annotations are the bridge.
12. [The App](12-the-app.md) — **what Claude Desktop looks like.** Every noun is a class. Every action is a method. Sidebar, conversation list, project cards, project conversations, composer, thinking block, response text. The app described as a user sees it, with the object mapping to code. This is the specification the code implements.
