# Coding Policy

Arthur: How the team writes code. Two audiences — framework developers (primarily Cathy, working in `chemistry/src/`) and framework users (Phillip, Gabby, working in `chemistry/app/` and `.public/`). Both need the code to be legible. $Chemistry should read like chemistry, not like Java.

Arthur: Created in sprint 41 when the team imported library infrastructure from dna-library. The original Coding Policy described PowerShell automation scripts and was archived. This one is born here.

## Chapters

1. [The $ convention](01-the-dollar-convention.md) — The `$` prefix separates what something IS from where it's USED. `$Chemical` is the framework concept (intrinsic); `$()` creates the React wrapper (extrinsic). `$derivatives$` and `$scope$` are framework-internal (double `$`). User state has no prefix: `this.count`, `this.reversed`. camelCase always — no ALL_CAPS constants, no UPPER_SNAKE. The `$` is a namespace without the ceremony.

2. [Reactive patterns](02-reactive-patterns.md) — Four patterns that shape all $Chemistry code. Scope-tracked reactivity: write `this.count = 5` and the view updates — no `.value`, no `setState()`. View purity: same state in, same output out — a mathematical guarantee, not an optimization. Safe composition: chemicals compose freely because views can't corrupt each other's state. The getter pattern: `get Card() { return DefaultCard; }` — visual identity as overridable getters, behavioral sub-components as methods. Template methods for subclass-overridable rendering.

3. [Framework vs application](03-framework-vs-application.md) — Three layers with different rules. `chemistry/src/` is the framework (Cathy's territory, maximum test coverage, `$()` as sole public surface, internal symbols symbol-keyed). `chemistry/app/` is the Lab (Phillip's territory, eat-our-own-dogfood, no raw React function components). `.public/` is the public face (the teaser page, deployed to GitHub Pages, beauty is a deliverable). Dependencies flow inward: .public → chemistry, app → src. The framework never knows about its applications.
