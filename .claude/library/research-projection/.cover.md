# Research Projection

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **coauthor:** [Adam](../..teamsmanship/..team/adam/adam-between-the-wires/.cover.md), [Claude](../..teamsmanship/..team/claude/claude-or-the-recursive-mirror/.cover.md)
- **subject:** [Sprints](../.projection/.cover.md)

---

The sprint-by-sprint record of building the Claude Desktop automation tool — the team's research instrument. Started when Doug's domain was lost and the team needed to preserve conversations from his Claude account. Evolved from PowerShell scripts through a stateful TypeScript application to a session-based conversation system. Now being lifted into this repo as the [Reference Desk](../reference-desk/.cover.md).

## Sprints

1. [Sprint 33 — The Foundation](01-sprint-33--the-foundation.md) — first connection to Claude Desktop. Prove UIA tooling, establish the object model that mirrors the app.
2. [Sprint 34 — Can We Talk to Claude?](02-sprint-34--can-we-talk-to-claude.md) — spike: get Claude Desktop open and interactable, or conclude we can't and document why.
3. [Sprint 35 — The Object Model](03-sprint-35--the-object-model.md) — TypeScript class hierarchy modeling the Claude app as a human sees it: screens, pages, components.
4. [Sprint 36 — The Ghost App](04-sprint-36--the-ghost-app.md) — build the full controller skeleton. Every class named and typed, bodies empty — the ghost of the app.
5. [Sprint 37 — The Archive](05-sprint-37--the-archive.md) — parse the Claude export into a browsable markdown library.
6. [Sprint 38 — Projects as Books](06-sprint-38--projects-as-books.md) — model Claude projects as library books with conversation chapters.
7. [Sprint 39 — The App Export](07-sprint-39--the-app-export.md) — drive Claude Desktop to capture project-conversation mappings iteratively.
8. [Sprint 40 — Smart Selectors](08-sprint-40--smart-selectors-and-lazy-data.md) — scoped selectors that understand page context, lazy async data properties.
9. [Sprint 41 — The Stateful App](09-sprint-41--the-stateful-app.md) — implement the app model design. Active page, navigation state machine, the app as product.
10. [Sprint 42 — Projects as Books](10-sprint-42--projects-as-books.md) — project content as structured library chapters with conversation summaries.
11. [Sprint 43 — Spring Cleaning](11-sprint-43--spring-cleaning.md) — archive dead code, dead skills, dead roles. Leave the repo honest about what's active.
12. [Sprint 44 — A Complete Project Book](12-sprint-44--a-complete-project-book.md) — one project book with everything: files, conversations, artifacts. Navigable for Doug.
13. [Sprint 45 — The Migration Plan](13-sprint-45--the-migration-plan.md) — what gets pushed to the new account, in what form, and how the library stays in sync.
14. [Sprint 46 — The Full Picture](14-sprint-46--the-full-picture.md) — run the infrastructure, get data current, look at the whole account and decide how to preserve it.
15. [Sprint 47 — Scaffolding](15-sprint-47--scaffolding-and-file-books.md) — wire project-conversation mapping in code, scaffold project books with conversation chapters.
16. [Sprint 48 — The First Twenty](16-sprint-48--the-first-twenty.md) — read conversations 1-20 chronologically. Build understanding of Doug, Ana, and Claude-chat.
17. [Sprint 49 — Eirian Arrives](17-sprint-49--eirian-arrives.md) — read conversations 21-40. The account transforms: Eirian, $Chemistry, the project's purpose emerges.
18. [Sprint 54 — The Transfer Protocol](18-sprint-54--the-transfer-protocol.md) — the protocol for pushing context to the new account, written as migration book chapters.
19. [Sprint 55 — TOC Summaries](19-sprint-55--toc-summaries--virtual-seren.md) — conversation summaries and index generation.
20. [Sprint 56 — App Implementation](20-sprint-56--app-implementation-instructions--migration.md) — the app reads and writes project content. Instructions and migration automation.
21. [Sprint 57 — Creating Projects](21-sprint-57--creating-projects-and-pushing-instructions.md) — create all projects on the new account, push instructions, upload files and covers.
22. [Sprint 58 — Writing Instructions](22-sprint-58--writing-real-project-instructions.md) — generate project instructions from library content, one project at a time.
23. [Sprint 59 — Create and Upload](23-sprint-59--create-projects-and-upload-files.md) — create projects and upload files via clipboard.
24. [Sprint 60 — Upload Covers](24-sprint-60--upload-conversation-history-covers.md) — upload conversation history covers to each project.
25. [Sprint 61 — Feedback and Hardening](25-sprint-61--feedback-mcp-research-and-app-hardening.md) — address blind spots: code reliability, architecture review, MCP research.
26. [Sprint 62 — Conversation Automation](26-sprint-62--conversation-automation.md) — build and verify real-time conversation capability. Test exhaustively, take screenshots of every UI state.
27. [Sprint 63 — The Pilot Conversation](27-sprint-63--the-pilot-conversation.md) — first live conversation with Claude Desktop. DNA Patternity project. Claude leads.
28. [Sprint 64 — The Conversation Object Model](28-sprint-64--the-conversation-object-model.md) — model conversations as objects: create, rename, delete, compose, stream, read turns.
29. [Sprint 65 — The Composed Message](29-sprint-65--the-composed-message.md) — the message being built is a stateful object reconstructable from the UIA tree.
30. [Sprint 66 — The Conversation Object](30-sprint-66--the-conversation-object.md) — Turn, Prompt, Response, Content, lazy Artifacts. Full conversation parsing.
31. [Sprint 67 — Conversation Sessions](31-sprint-67--conversation-sessions.md) — session lifecycle: start, send, read, end. The tool becomes interactive.
32. [Sprint 69 — Sequence Filtering](32-sprint-69--sequence-filtering-paper-synthesis.md) — paper synthesis using the tool. Research applied to neuroscience content.
