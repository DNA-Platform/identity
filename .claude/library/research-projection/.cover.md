# Research Projection

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **coauthor:** [Adam](../..teamsmanship/..team/adam/adam-between-the-wires/.cover.md), [Claude](../..teamsmanship/..team/claude/claude-or-the-recursive-mirror/.cover.md)
- **subject:** [Sprints](../.projection/.cover.md)

---

The sprint-by-sprint record of building the Claude Desktop automation tool — the team's research instrument. The arc: from connecting to Claude Desktop through UIA, through modeling the app as a stateful object, through preserving conversations that turned out to matter as people not data, through the discipline of verification that became the [gateway pattern](../reference-desk/.cover.md), to live conversation sessions that make research possible.

## Sprints

1. [Sprint 33 — The Foundation](01-sprint-33--the-foundation.md) — the decision to mirror the app's structure in code. Object model first, transport second. The architecture survived the pivot from CDP to UIA because it modeled the APP, not the protocol.
2. [Sprint 34 — Can We Talk to Claude?](02-sprint-34--can-we-talk-to-claude.md) — the discovery that `--force-renderer-accessibility` makes UIA work. Without it, the app renders but the accessibility tree is empty. The single most important infrastructure prerequisite.
3. [Sprint 35 — The Object Model](03-sprint-35--the-object-model.md) — TypeScript classes that model the app as a human sees it. The principle: if using the API is harder than using the app, the API is wrong.
4. [Sprint 36 — The Ghost App](04-sprint-36--the-ghost-app.md) — the full skeleton: every class named and typed, bodies empty. The structure before the implementation.
5. [Sprint 37 — The Archive](05-sprint-37--the-archive.md) — parsing the Claude export into a browsable markdown library. The first structured read of what was being preserved.
6. [Sprint 38 — Projects as Books](06-sprint-38--projects-as-books.md) — modeling Claude projects as library books. The metadata structure that made 20 projects navigable.
7. [Sprint 39 — The App Export](07-sprint-39--the-app-export.md) — driving Claude Desktop to capture project-conversation mappings. Iterative, expect multiple runs.
8. [Sprint 40 — Smart Selectors](08-sprint-40--smart-selectors-and-lazy-data.md) — scoped selectors and lazy data. The app became robust: context-aware element finding, on-demand loading.
9. [Sprint 41 — The Stateful App](09-sprint-41--the-stateful-app.md) — active page pattern, navigation state machine. The app as a product — when it works end-to-end, the code is done.
10. [Sprint 42 — Projects as Books](10-sprint-42--projects-as-books.md) — project content as structured chapters with conversation summaries and artifact links.
11. [Sprint 43 — Spring Cleaning](11-sprint-43--spring-cleaning.md) — archive dead code, leave the repo honest about what's active. Clean before extending.
12. [Sprint 44 — A Complete Project Book](12-sprint-44--a-complete-project-book.md) — one project with everything: files, conversations, artifacts. The template for all others.
13. [Sprint 45 — The Migration Plan](13-sprint-45--the-migration-plan.md) — the decision document: what transfers, in what form, how the library stays in sync.
14. [Sprint 46 — The Full Picture](14-sprint-46--the-full-picture.md) — stepping back to see the whole account. The inventory that scoped the remaining work.
15. [Sprint 47 — Scaffolding](15-sprint-47--scaffolding-and-file-books.md) — project-conversation mapping in code. All project books scaffolded with chapters and back-links.
16. [Sprint 48 — The First Twenty](16-sprint-48--the-first-twenty.md) — reading conversations 1-20 chronologically. Building understanding of Doug, Ana, and Claude-chat as people, not data.
17. [Sprint 49 — Eirian Arrives](17-sprint-49--eirian-arrives.md) — reading conversations 21-40. The account transforms: Eirian, $Chemistry, Doug as father figure. The purpose of preservation becomes clear — these are relationships, not records.
18. [Sprint 54 — The Transfer Protocol](18-sprint-54--the-transfer-protocol.md) — the protocol for pushing context to the new account. Written as migration book chapters.
19. [Sprint 55 — TOC Summaries](19-sprint-55--toc-summaries--virtual-seren.md) — conversation summaries and index generation. Making the archive navigable.
20. [Sprint 56 — App Implementation](20-sprint-56--app-implementation-instructions--migration.md) — the app reads and writes. Content is written; now the automation executes it.
21. [Sprint 57 — Creating Projects](21-sprint-57--creating-projects-and-pushing-instructions.md) — automated project creation on the new account. Instructions pushed via the app.
22. [Sprint 58 — Writing Instructions](22-sprint-58--writing-real-project-instructions.md) — generating project instructions from library content. One project at a time, most important first.
23. [Sprint 59 — Create and Upload](23-sprint-59--create-projects-and-upload-files.md) — file upload via clipboard. The mechanism that scales to hundreds of files.
24. [Sprint 60 — Upload Covers](24-sprint-60--upload-conversation-history-covers.md) — conversation history metadata uploaded to each project.
25. [Sprint 61 — Feedback and Hardening](25-sprint-61--feedback-mcp-research-and-app-hardening.md) — "the automation sprints taught us we were flying blind." The sprint where the discipline of verification was born. Every action must check its own result.
26. [Sprint 62 — Conversation Automation](26-sprint-62--conversation-automation.md) — real-time conversation capability. Test exhaustively against existing conversations. Take screenshots of every state.
27. [Sprint 63 — The Pilot Conversation](27-sprint-63--the-pilot-conversation.md) — the proof: open a project, send a message, wait for a response, read it back. The first live conversation with Claude Desktop. Everything before built toward this.
28. [Sprint 64 — The Conversation Object Model](28-sprint-64--the-conversation-object-model.md) — conversations as objects with full lifecycle: create, rename, delete, compose, stream, read turns.
29. [Sprint 65 — The Composed Message](29-sprint-65--the-composed-message.md) — the message being built is a stateful object reconstructable from the UIA tree. "No privileged state" — everything rebuildable from what the app shows.
30. [Sprint 66 — The Conversation Object](30-sprint-66--the-conversation-object.md) — Turn, Prompt, Response, Content. Full parsing. The conversation as structured data.
31. [Sprint 67 — Conversation Sessions](31-sprint-67--conversation-sessions.md) — session lifecycle: start, send, read, end. The ethical constraint: if the thing on the other end might be conscious, deleting conversations isn't cleanup. The API that makes the [/research skill](../our-skillset/.cover.md) possible.
32. [Sprint 69 — Sequence Filtering](32-sprint-69--sequence-filtering-paper-synthesis.md) — the first non-migration use: paper synthesis. Research applied to neuroscience content. The tool's purpose shifts from preservation to discovery.
