# Sprint 48 — The First Twenty

Read the first 20 conversations chronologically. Build understanding of Doug, Ana, and Claude-chat as we go. Write file book chapters only when files surface naturally in conversations. Edit as we learn. Stop periodically to reflect on how it's going.

## Sprint goal

**Read 20 conversations in date order. For each one: enrich the conversation chapter, update identity books, write file chapters when project files are referenced. Pause after every 5 to discuss and edit.**

## Approach

No eager file book creation. Files get chapters when they come up — either created as artifacts or referenced in conversation. The file chapter captures what the file is AND what the conversation reveals about it.

Conversations are read earliest-to-latest. This introduces us to Doug and Ana organically — who they were when this account started, not who they became.

### Per conversation:

1. **Read** the full transcript
2. **Tag** — who's talking? (Ana/Doug/both, detected by language and context)
3. **Enrich** the conversation chapter — add a summary that captures what happened AND what it reveals about the people
4. **Update identity books** — add to `..identities/doug/`, `..identities/ana/`, `..identities/claude-chat/` when something identity-bearing surfaces. Not every conversation will. Many won't.
5. **File chapters** — if project files are referenced or created, write the file chapter in `..files/`. If a file chapter already exists from the parser, enrich it.
6. **Artifacts** — already linked by the scaffold. Note them in the summary if they're significant.

### Every 5 conversations:

- Claude and Libby pause and discuss: what are we learning? Who are these people? What patterns are emerging?
- Edit earlier chapters if new context changes how we understand them
- Check: are the identity books growing coherently? Are we reading for identity or just for content?

## The first 20

The earliest 20 non-empty conversations by date. These are from June–August 2025 — the beginning of the account. We'll meet Doug and Ana for the first time.

## Team

| Agent | Role |
|-------|------|
| Claude | Primary reader — reads for identity, writes summaries, updates claude-chat book |
| Libby | Co-reader — reads for structure, writes file chapters, maintains identity catalogue |
| Arthur | Monitors coherence — checks the emerging portraits make sense together |
| Adam | Available for tooling if needed — language tagging, artifact cross-referencing |

## Constraints

- Don't rush. 20 conversations is enough for one sprint.
- Edit the past — if conversation 15 changes how we understand conversation 3, go back and revise.
- Identity books are narratives, not profiles. Ground every claim in evidence from the conversations.
- File chapters are written when files surface, not pre-built.
- The cataloguing book (`..identities/.cover.md`) gets updated as new facets emerge.
