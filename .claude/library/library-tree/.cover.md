# Library Tree

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **subject:** [Knowledge](../..librarianship/.cover.md)

---

The library is a tree with a main branch and project branches. The main branch is the identity — the team's shared knowledge at `.claude/library/`. Project branches are `.lib/` directories that live beside the code they record the team's learning about. This book specifies how the tree grows: what a branch is, how it catalogues itself, how sprints project team effort onto a project, and how to set up a new branch.

A branch is NOT code documentation. A branch is the team's record of applied knowledge — collaboration, learning, decisions. The team's memory of building a framework, written in library conventions. A `.lib/` beside a framework records what the team learned building it, not what the framework does. The framework's API reference belongs in subject catalogues. The team's journey belongs in the branch.

## Chapters

1. [Branches](01-branches.md) — `specification: Branch`. What a branch is, the `.lib/` convention, the one-way link rule, placement. The branch as the team's record of applied knowledge. Also catalogues known branches: [$Chemistry](../../library/chemistry/.lib/..representivity/.cover.md) at `library/chemistry/.lib/`.

2. [Cataloguing](02-cataloguing.md) — `specification: Catalogue`. Each branch has a `..` prefixed cataloguing book — the branch's identity. The name carries meaning about what the branch represents. The cataloguing book organises the branch's knowledge the way [Librarianship](../..librarianship/.cover.md) organises the main library's.

3. [Sprints](03-sprints.md) — `specification: Sprint`. How sprints are represented in a branch. The sprint book is called Projection — a projection of team effort onto the project. Sprints are chapters in Projection, chronological, the branch's autobiography.

4. [Setup](04-setup.md) — How to set up a new branch. Step by step: create `.lib/`, create the cataloguing book, create Projection, add territory, catalogue the branch here.

5. [Known Branches](05-branches.md) — Catalogue of every known branch. Each entry: repo, location, cataloguing book, sprint book. Currently: [$Chemistry](../../library/chemistry/.lib/..representivity/.cover.md) at `library/chemistry/.lib/` in inexplicable-phenomena.
