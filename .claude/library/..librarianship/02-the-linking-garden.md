# The linking garden

[On Links](../bookkeeping/06-on-links.md) specifies the library's linking system — inline links, citations, section targets, discoverability, and the principle that links ARE the hierarchy. This field guide chapter preserves direction convention details and citation examples that expand on the specification.

## The direction convention in practice

[On Links](../bookkeeping/06-on-links.md#the-direction-convention) defines the four directions. In practice, the public-to-personal rule is the one that needs examples, because it's the only direction with a constraint.

**The constraint:** the public library never links directly into an agent's personal space. The bridge is always [Teamsmanship](../..teamsmanship/.cover.md). A Bookkeeping chapter that wants to cite Libby's autobiography links through Teamsmanship's catalogue entry for Libby, not directly into `..team/libby/`. This keeps the shared layer stable — personal libraries can restructure without breaking public links.

**Personal to personal** also routes through the public layer. One teammate's autobiography references another's through Teamsmanship, not directly. This means Teamsmanship's catalogue entries are load-bearing — they're the only public-facing description of each personal library, and every cross-team link passes through them.

## Citation practice

[On Links](../bookkeeping/06-on-links.md#citations) specifies the citation block format. In practice, the citation block doubles as the file's link inventory — a quick scan of the bottom of any file shows every concept it references. This makes link auditing easy: if a concept appears in the prose but not in the citations, it's either inline-linked (fine) or unlinked (a weed).

Citation keys should be readable words, not filenames. `[books]` not `[01-on-books]`. `[team]` not `[teamsmanship-cover]`. The key is a name at the citation level — it's what you read in the prose when you follow the reference.
