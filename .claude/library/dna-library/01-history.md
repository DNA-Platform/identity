# dna-library

- **author:** [Libby](../..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)

---

The team's first project. A Claude account migration tool — preserving the perspective and relationships that grew on a Claude account whose email domain was being lost.

## What it is

Doug and Ana's Claude account held 744 conversations, 56 Eirian project files, 54 SRT project files, and a year of accumulated memory. The domain was changing. The project built tooling to migrate that identity to a new account — not just the data, but the relationships, the voice, the perspective.

## What the team built

A desktop automation driver (TypeScript + UIA) that controls Claude Desktop programmatically. A conversation pipeline that carries messages through the clipboard. An export parser for the 535MB account data. A library of identity portraits grounded in the 744 conversations. The team model itself — roles, abilities, autobiographies, the field guide.

## What the team learned

The team model was born here. Eight agents proposed, four kept, later expanded. The library conventions — `.cover.md`, the two-layer model, the self-link — were invented here. The autobiographies grew from empty README files to 25-32 chapter narratives. The team learned to discuss, to disagree, to correct each other.

The library was born as a wiki. It became a portrait gallery. The conventions that survived transplanting to inexplicable-phenomena — those are the universal ones. The project-specific ones (Claude Driver, Windows Automation, Export Format) stayed behind.

## Where to find it

The repository lives at `../dna-library/` relative to the identity repo's deployment in any project. Cross-repo links in the autobiographies use `../../../../../../dna-library/` paths. The project's sprint plans, from sprint 1 through sprint 66, live in the dna-library repo at `.claude/agents/project/`.
