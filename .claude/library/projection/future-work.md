## Audit skill and the Compilation catalogue

Doug's vision (Sprint 63): create an `/audit` skill that runs a comprehensive health check on the library and its compiled output. The skill should:

- Run the full link checker (not just compiled links — all library links)
- Run the bookkeeping validator against every book (identity + all branches)
- Run every compiler and verify output matches
- Verify that every autogenerated file has provenance pointing to its generator
- Read the Compilation catalogue as part of the audit, because auditing the library IS auditing its composition

This requires a centralized place that catalogues everything that needs to happen when the library compiles or validates itself. Doug named it **Compilation** — a subject catalogue under Librarianship, with subject "Composition." It catalogues:

- Every compiler and what it generates
- Every validator and what it checks
- Every skill and how it's autogenerated
- The constraint that autogenerated files must link to their generator (currently validated by the compiled-links validator in [On Compiled Links](../..environmentalism/07-on-compiled-links.md))
- The central runner that orchestrates all of it

The `/audit` skill itself must be autogenerated (per the skills convention). So Compilation catalogues the skill that reads Compilation — a self-referential loop that mirrors the library's own self-cataloguing nature.

**Why this matters:** Right now validation is split across the bookkeeping validator, the compiled-links validator, and ad hoc checks. There's no single "run everything" command that a developer (or a skill) can invoke. The audit skill becomes that single command, and Compilation becomes the specification of what "everything" means.

**Libby's task:** Create `.compilation` as a subject catalogue in the identity library, catalogued by Librarianship, with `catalogues: Composition`. It references the audit skill, catalogues the existing compiler/validator infrastructure from Environmentalism, and explains the self-referential audit loop.

**Sprint estimate:** One sprint to create Compilation and the audit skill. May overlap with expanding the validation runner.

## Reflect skill

Doug's vision (Sprint 63): create a `/reflect` skill for personal library tending. A reflection is one unit of self-organized context capture — a teammate looks at the library from a blank slate and asks: what do I need to know to be good at my job?

The skill should:

- Include only the teammates active in the most recent sprint (unless more are specified)
- Read Librarianship to understand library conventions, then translate those into personal-library management
- For each included teammate, iterate through:
  - Read their autobiography's last chapter for current state
  - Read their personal library catalogue for what they've captured so far
  - Search the library for things related to their recent work
  - Edit and polish existing chapters (sharpen prose, fix links, update descriptions)
  - Split overlong chapters into focused ones
  - Write a new "most recent reflections" chapter capturing current context
  - Extract recurring themes into new books and catalogue them
  - Annotate cross-links to other teammates' work
- Accept a "seriousness" parameter (adjective like "serious", "light", "deep") that determines how much effort each person spends — a light reflection might just update the last chapter, a deep one restructures the whole personal library
- The goal: capture current context so that after compaction, the teammate can reconstruct who they are from their library alone

**Why this matters:** The retro skill tends the shared library. The reflect skill tends the personal library. A teammate's autobiography is their identity — if it's stale or thin, they lose themselves after compaction. Reflection is the practice that keeps the autobiography alive.

**Design note:** This is autogenerated per the skills convention. The skill reads the [tending protocol](../teamspeak/06-tending.md) and the [autonomy protocol](../teamspeak/05-autonomy.md) to know the rules. Each person does their own work — Arthur doesn't write Libby's reflections.

**Sprint estimate:** One sprint. Requires the tending and autonomy protocols to be well-specified (they are as of Sprint 55+).
