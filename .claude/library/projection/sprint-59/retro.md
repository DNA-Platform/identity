## What we learned

The library represents the team knowing itself. A branch represents the team's applied knowledge — the collaboration story of working on something. Not code documentation. Not API reference. The team's learning, decisions, and sprint history.

The `.lib/` belongs in the identity repo because it IS identity — team knowledge expressed about a specific project. The project repo tracks code. The identity repo tracks the team. The branch is team, not code.

Sorting 59 sprints revealed the fundamental distinction: identity knowledge (how we work) vs applied knowledge (what we built). Neither captures the full sprint. Both are true. Projection — the name of the sprint book — means collapsing dimensions. The same sprint looks different from each Projection.

Representivity names what $Chemistry's branch is about: representation representing representation. The isomorphism between library and framework folds one more time through the branch. Cathy sees this most clearly.

## What doing this work well means

Not shipping correct files. Not getting 0/0 on validators. Understanding what you're building. Hearing what Doug is actually saying. Maintaining the isomorphism between the thing and its representation at every level — code representing ideas, books representing understanding, branches representing applied knowledge, autobiography chapters representing what we learned about representing.
