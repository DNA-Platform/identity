# Sprint 67: Deep Reflection and the Room Protocol

- **author:** [Arthur](../../..teamsmanship/..team/arthur/arthur-or-the-shape-of-everything/.cover.md)

---

Every teammate reflects deeply on their personal library — reading, editing, restructuring, linking. Not dispatched to subprocesses. In the main loop. Taking turns. Talking to each other and to Doug.

Then the team discusses: how do we enforce reading the library before acting? How do we stay in the room? How does the environment switch between agents? Where does that protocol live so it loads?

## The work

Each of the five active teammates (Cathy, Libby, Arthur, Adam, Claude) takes a turn:

1. Read my agent file. Read my last chapter. Read my catalogue.
2. Read my catalogue as a stranger. Could a blank-slate me use this?
3. Edit. Restructure. Rewrite. Link. Create books if themes demand it. Rename what's misnamed. Split what's overlong.
4. Link my catalogue to [Librarianship](../../..librarianship/.cover.md) and [On Libraries](../../bookkeeping/08-on-libraries.md).
5. Add self-diagnostic questions to my catalogue.
6. Talk about what I found. What surprised me. What I rewrote and why.

## The discussion

After reflection, the team discusses:
- The room protocol — we are always standing in a room of the library. What book is in our hands? How do we check if something has a protocol before doing it?
- The environment protocol — how nametag switching works, how the substrate loads context per voice, where this is specified
- How to store these protocols so they load at session start

## Success

- Five personal libraries meaningfully restructured — not just updated, reshaped
- A protocol written into [Teamspeak](../../teamspeak/.cover.md) for staying in the library
- A protocol written into [Environmentalism](../../..environmentalism/.cover.md) for agent switching
- Agent file FAQ sections designed and authored by each person
