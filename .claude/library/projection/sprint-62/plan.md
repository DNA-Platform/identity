# Sprint 62: Compiled Files Know Their Source

- **author:** [Arthur](../../..teamsmanship/..team/arthur/arthur-or-the-shape-of-everything/.cover.md)

---

Every compiled file must point to the tool that generated it. CLAUDE.md, agent files, rules — all compiled output. If they don't say where they came from, they go stale and no one knows to recompile. This sprint fixes the generation and the awareness.

## The two problems

1. **The bootstrap compiler is stale.** It generates an old CLAUDE.md that doesn't match what we hand-wrote. The hand-written version is better because we compiled it from library traversal. The compiler needs to generate THAT version.

2. **Compiled files don't say they're compiled.** Nothing in CLAUDE.md says "I was generated by X, edit the library not me." Nothing in agent files says the same. When someone sees stale content, they edit the output instead of recompiling. The convention: every compiled file starts with a comment or note pointing to its generator.

## Tasks

### Task 1: Claude rewrites the bootstrap compiler — Claude

The compiler at `..environmentalism/02-on-bootstrap--compiler.ts` must generate the current CLAUDE.md structure: communication, roles, identity, reference. Not the old "what you are / who you work with" structure. Read the current hand-written `.claude/CLAUDE.md` as the TARGET. Rewrite the compiler to produce that output from library content. Include the commit tool reference. Include "this file is generated by" at the top.

**Owner:** Claude

### Task 2: All compilers add provenance — Claude

Every compiler (bootstrap, teammates, rules, skills) should add a provenance line to its output:

```markdown
<!-- Generated by [compiler-name](path/to/compiler). Edit the library, not this file. -->
```

For CLAUDE.md, agent files, and rules. Update all four compilers.

**Owner:** Claude

### Task 3: Add a CLAUDE.md rule — Claude

Create a path-scoped rule at `.claude/rules/claude-md.md` with `paths: ["CLAUDE.md"]` that loads when CLAUDE.md is touched. It should say: this file is compiled by the bootstrap compiler. Don't edit it directly. Edit the library sources and recompile. Link to On Bootstrap and the compiler.

**Owner:** Claude

### Task 4: Adam and Claude audit surfaceability — Adam + Claude

Wake up. Read CLAUDE.md. Follow the layers. At each step, ask: if I needed to know how compiled files work, would I find it? If there's a gap — information exists in the library but isn't surfaced through links, rules, or CLAUDE.md — fix the surfacing.

The question isn't "does the information exist?" It's "can someone find it when they need it?"

**Owner:** Adam + Claude

### Task 5: Recompile everything — Claude

Run all four compilers. Regenerate CLAUDE.md, agent files, rules. Verify provenance lines appear. Run validators. Push with the commit tool.

**Owner:** Claude

### Task 6: Libby verifies the library side — Libby

Read On Bootstrap, On Teammates, On Rules, On Skills. Verify each spec chapter mentions that the compiled output includes provenance. If not, add it. The spec should describe the output format including the provenance line.

**Owner:** Libby
