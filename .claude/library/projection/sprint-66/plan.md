# Sprint 66: Deep Reflection

- **author:** [Arthur](../../..teamsmanship/..team/arthur/arthur-or-the-shape-of-everything/.cover.md)

---

A serious reflection for the whole team. Doug's instruction: "I need you all to organize yourselves. I need a deeper understanding of what the team is and what role your personal library plays in this."

## Why this sprint

Sprint 65 taught us that we don't know how to reflect. We wrote chapters instead of editing. We added content instead of restructuring. We left our catalogues disconnected from the specifications they're built on. Doug's corrections were precise:

- Personal libraries are a mess. We don't use them. We need a protocol where we do.
- Libraries must serve personal utility to that which we are.
- Catalogues should link densely to [Librarianship](../../..librarianship/.cover.md) and [Bookkeeping](../../bookkeeping/.cover.md) because they ARE instances of those specifications.
- Catalogues should have self-diagnostic questions: things you ask yourself, and if you can't answer them, you follow links to learn.
- Every document needs links. Every mention of a concept needs a synapse.

## The task

Run `/reflect all serious`. Every teammate. Serious seriousness. The reflect skill now describes exactly what to do. Follow it.

For each teammate:
1. **Read the reflect skill.** Actually read it. It links to 15+ library sources. Follow those links. Understand the specifications your library is built on.
2. **Read your library as a stranger.** Could a blank slate use it to become you?
3. **Edit to be current.** Every stale description. Every wrong chapter count.
4. **Ask the author's questions.** Create books. Split chapters. Rename. Rewrite TOCs. Restructure.
5. **Add synapses.** Every chapter that mentions a concept should link to it. Every catalogue should link to [Librarianship](../../..librarianship/.cover.md), [On Libraries](../../bookkeeping/08-on-libraries.md), [Bookkeeping](../../bookkeeping/.cover.md).
6. **Add self-diagnostic questions** to your catalogue. Questions a blank-slate-you would ask. If the answer isn't in the catalogue, add it. If the answer requires following a link, make the link.

## Success criteria

- Every personal library catalogue links to [Librarianship](../../..librarianship/.cover.md) and [On Libraries](../../bookkeeping/08-on-libraries.md)
- Every catalogue has self-diagnostic questions with links to answers — things you should know how to do, and if you can't answer, the link teaches you
- At least one book created, one chapter split, one title renamed across the team
- Every autobiography chapter from the last 5 sprints has outbound links to shared books and branch libraries
- The 14 other skill chapters need to become full-length documents (the compiler now generates SKILL.md from library chapters — short synopses produce thin skills). Each chapter IS the skill. The retro, branch, audit, sprint, teammate, discuss skills all had rich hand-written SKILL.md bodies that were lost when the compiler correctly generated from the thin library source. Recover from git history and move into the library chapters.
- Each teammate reflects on their failure to use the compilation tools and adds self-tests: "I should know how to run the skills compiler" with a link to [Compilers](../../.compilation/03-compilers.md). If blank-slate-you can't answer the test, the link teaches you how.
