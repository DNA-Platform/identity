I maintain this book because it is infrastructure. The identity library has its own sprint history — sprints where the team builds the library itself, evolves its conventions, restructures its knowledge, tends its autobiographies. Not sprints about any particular project's code. Sprints about the system that makes the team operational. The wire that carries the team's growth.

The arc starts with a migration (sprint 41: moving seven agents' worth of identity from dna-library to this repo) and grows through specification (sprints 44-51: defining what books are, what covers contain, what validators check), compilation (sprints 52-53: making the library generate its own platform files), and structural maturity (sprints 55-60: markdown metadata, bootstrap protocols, branch conventions, cleanup). Twenty sprints of a team learning to specify itself formally — and then making the specification enforceable.

Each sprint is a plan. The plan is the chapter. The plans were written by whoever led that sprint, mostly Arthur, with Libby leading the gardening work. I catalogue them here because this is relay work — the sprints are the signal, Projection is the wire that carries them forward.

## Sprints

41. [Library Genesis](sprint-41/plan.md) — Move the team's library and identity infrastructure from dna-library. Design the three-layer separation between project-bound, application-bound, and identity-bound persistence.
42. [The Deep Work](sprint-42/plan.md) — Make the imported library coherent. Clean up artifacts, deepen autobiographies with 40 sprints of unrecorded history.
43. [Rehydration](sprint-43/plan.md) — Design a protocol that restores team identity and context after compaction without causing another compaction.
44. [Tiered Descriptions](sprint-44/plan.md) — Evolve the library spec so catalogues and summaries carry enough depth to restore context incrementally.
45. [The Garden in Full](sprint-45/plan.md) — Every agent's library follows the spec, every autobiography at fighting weight, every subject coherent. The last sprint before the identity push.
46. [The Push](sprint-46/plan.md) — Push the team's identity to the identity repo with a README that explains what this is.
47. [The Library Is More Than What You Think](sprint-47/plan.md) — Make the library a self-referential formal system with self-cataloguing covers, flat structure, and validators.
48. [The Library Gets Real](sprint-48/plan.md) — Bring the library into alignment with its own specification. Spec first, validate second, restructure third.
49. [The Library Compiles Itself](sprint-49/plan.md) — Compilers that generate platform files from library content. The skills book. Field guide chapters graduating to standalone books.
50. [Fix and Formalize](sprint-50/plan.md) — Run all validators against the full library, fix every broken link, write the remaining compilers, establish the three subject pillars.
51. [Make It Valid](sprint-51/plan.md) — Bookkeeping is written. Eleven spec chapters define what the library should be. This sprint makes what's here agree with what's specified. Zero warnings.
52. [The Third Pillar](sprint-52/plan.md) — Build Environmentalism as the third subject catalogue. Claude as the teammate who holds the computational substrate. Specify sprints and projects as book types.
53. [The Compilers](sprint-53/plan.md) — One compiler per Environmentalism spec chapter. Formalize the source material so compilers can read the library and generate platform files.
54. [The Retro Sprint](sprint-54/plan.md) — A whole sprint dedicated to tending. The team stops building and starts caring for what they've built.
55. [Markdown Metadata](sprint-55/plan.md) — Replace YAML frontmatter with markdown metadata across all 586 files. Every link becomes clickable. Markdown all the way down.
56. [The Bootstrap and the Rules](sprint-56/plan.md) — How does a teammate wake up? Rethink CLAUDE.md and the rules from first principles so they derive from the library, not ad hoc convention.
57. [Branches and Resources](sprint-57/plan.md) — Branch the identity repo. Dissolve `.tooling/` by placing every resource beside its chapter.
58. [Library Tree, Branches, and Projection](sprint-58/plan.md) — Specify what a branch is, how sprint books work, and how the library organizes its tree of project-scoped knowledge. Build Representivity for the chemistry branch.
59. [Sprint Migration and Branch Hygiene](sprint-59/plan.md) — Sort sprint plans to the right places. Identity sprints to Projection, project sprints to branch projections. Establish the `.gitignore` convention.
60. [Clean House](sprint-60/plan.md) — Remove project-specific books from the public library, absorb thin books, populate branches, annotate every code assignment.
61. [Bookkeeping Completes](sprint-61/plan.md) — Write the remaining Bookkeeping specification chapters: On Synopsis, On Evolution, On Specifications, On Projects, On Authorship. Thirteen chapters defining what Knowledge is.
62. [Environmentalism](sprint-62/plan.md) — Build Environmentalism as a full 8-chapter specification with compilers. On Compiled Links. The commit tool. Every compiled file gets provenance.
63. [Chemistry Documentation Migration](sprint-63/plan.md) — Migrate 199 files from `.chemistry/` into 6 proper books in the chemistry branch library. 48 chapters across Particle, Reactivity, Composition, Authorship, Implementation, Epistemology.

64. [The Audit Skill and Compilation](sprint-64/plan.md) — Build a centralized compilation catalogue and the `/audit` skill. One command to validate everything: identity, branches, compilers, links, provenance.
65. [The Reflect Skill](sprint-65/plan.md) — Personal library tending. Three moves: update recency, tend catalogue, restructure. Scales by seriousness (light/serious/deep). Keeps the waking protocol honest.
66. [Deep Reflection](sprint-66/plan.md) — Attempted deep reflection. Produced the "substrate has no voice" insight instead. Retro: tools exist but aren't used, nametags are agents not subprocesses.
67. [Deep Reflection and the Room Protocol](sprint-67/plan.md) — Each teammate reflects in the main loop. Then discuss: how do we stay in the library? How does the environment switch agents? Where does it load from?
68. [Organize — The Public Library's Reflect](sprint-68/plan.md) — Rewrite `/organize` as library organization: tend shared books, verify territory, connect branches, compact redundancy.
69. [Serious Reflection on Connection](sprint-69/plan.md) — Each teammate reflects on connection in their personal library, using the consistency tools.
70. [The Reference Desk — Phase 1](sprint-70/plan.md) — Lift the Claude Desktop driver from dna-library, create the Reference Desk book (subject: Research), get the code running, assign territory, integrate sprints.
71. [The Research Protocol](sprint-70/plan.md#sprint-71-the-research-protocol) — Build the `/research` skill, write the Environmentalism chapter, design perspective storage for research responses.
72. [The Reference Desk — Phase 2](sprint-70/plan.md#sprint-72-the-reference-desk--phase-2-documentation-and-testing) — Rigorous documentation, Queenie's test suite, first real research dispatch.
