# Testing

- **author:** [Cathy](../../../../.claude/library/..teamsmanship/..team/cathy/cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [$Chemistry](../..representivity/.cover.md)

---

The test suite is one of the most substantial artifacts in $Chemistry — 428 tests across 48 files, organized into 7 directories covering abstraction, implementation, React integration, regression, specimens, and spikes. This book documents the testing practice: what the suite promises, how it's organized, what it costs at runtime, and how specific bugs were caught and pinned.

Epistemology asks "how do we know $Chemistry works?" This book answers with the harness: here are the tests, here's how they're structured, here's what they cost, here's how they catch regressions. The test suite is not documentation — it's a specification written as assertions. Reading the test descriptions is one of the fastest ways to learn what the framework guarantees.

## Chapters

1. [The Contract](01-the-contract.md) — what the test suite promises. 428 tests pin invariants. A behavior with no test is unpinned. The three roles: regression detection, specification crystallization, future-developer documentation.
2. [Organization](02-organization.md) — how tests are structured. Seven directories: `abstraction/` (particle, chemical, molecule, atom), `implementation/` (scope, reflection, promise), `react/` (integration, augmentation, rendering), `regression/`, `specimens/`, `spikes/`, `env/`. Naming conventions. What goes where.
3. [The Performance Contract](03-the-performance-contract.md) — what the framework costs at runtime. Two regimes: initial render (~170μs per chemical) and per-event re-render (essentially flat regardless of tree size). Granularity tradeoffs. Ship criteria. Known cost tradeoffs vs MobX, Vue, Solid.
4. [Specimens](04-specimens.md) — the Lab specimens that bridge testing and demonstration. Each specimen is a running example that cross-links to unit tests. The Lab is witness; the test suite is guarantee. `[SCAFFOLD]`
5. [Regression Stories](05-regression-stories.md) — how specific bugs were caught and pinned. Cross-chemical handler fan-out (sprint 24), short prop name instability (sprint 24), particularization prototype loss (sprint 22). Each story: the bug, the investigation, the fix, the test that pins it. `[SCAFFOLD]`
