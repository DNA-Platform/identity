# Reactivity

- **author:** [Cathy](../../../../.claude/library/..teamsmanship/..team/cathy/cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [$Chemistry](../..representivity/.cover.md)

---

I am the engine that makes representation live. Without me, a chemical is a static object with a view method — interesting but inert. With me, a chemical's state changes propagate through the system: writes trigger re-renders, scope tracks what was read, derivatives fan out, and the view updates because the aboutness relationship is real. I describe the reactive property model that I rebuilt from scratch in sprint 18 with scope-tracked getter/setter interception, and refined through sprints 19, 22, and 24 until it could handle cross-chemical writes, collection mutation, and per-instance bond accessors. This is the hardest part of the framework. It is also the part that makes the framework worth building.

## Chapters

1. [Reactive Properties](01-reactive-properties.md) — the `$x` convention, class field syntax, get/set accessor installation. Why mutation triggers re-render without `setState`. The `$Molecule` that installs accessors and the `$Reaction` that schedules updates.
2. [Scope Tracking](02-scope-tracking.md) — event handlers wrapped via `augment` run inside `withScope`. Reads recorded for snapshot. Writes recorded as dirty. `scope.finalize()` fires reactions and fans out to derivatives. The scope is the aboutness relationship made structural.
3. [Cross-Chemical Writes](03-cross-chemical-writes.md) — writing to another chemical's reactive property from inside a handler. The fan-out symmetry between in-scope and no-scope paths. The `hasOwnProperty($derivatives$)` ownership gate that prevents sibling leaks. The sprint-24 fix that made this work.
4. [Collection Mutation](04-collection-mutation.md) — Map/Set/Array methods detected via `$symbolize` snapshot diff. In-place mutation without replacement. The deterministic serialization that makes before/after comparison possible.
5. [Diffuse](05-diffuse.md) — the fan-out function in `scope.ts`. Walks `$derivatives$` only when own. The mechanism that connects a chemical's state change to every derivative that needs to re-render.
6. [Decorators](06-decorators.md) — `@inert()` opts out for `$x` properties that should not be reactive. `@reactive()` opts in for non-`$` properties that should be. `$Reflection`'s decorator registries and per-instance reactivity decisions.
7. [Glossary](07-glossary.md) — alphabetical reference of reactivity terms: Bond, Molecule, Reaction, `$isBound$`, diffuse, scope, derivatives.
