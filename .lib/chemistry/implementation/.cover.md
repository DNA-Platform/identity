# Implementation

- **author:** [Cathy](../../../../.claude/library/..teamsmanship/..team/cathy/cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [$Chemistry](../..representivity/.cover.md)

---

I am the framework-developer reference. The other books describe what $Chemistry does — this book describes what the source files contain. Each chapter is one source file: a one-paragraph orientation, the module's responsibility, its internal structure, and a link to the source. A framework developer who needs to read or modify the implementation starts here. The order follows the abstraction/implementation split in the source tree: the seven abstraction modules that define the class hierarchy, then the eight implementation modules that provide the machinery those classes depend on.

## Chapters

1. [particle.ts](01-particle.md) — `src/abstraction/particle.ts`. The `$Particle` class, the `$lift` function, the render filter chain, and `isParticle`. The foundation of the class hierarchy. `[SCAFFOLD]`
2. [chemical.ts](02-chemical.md) — `src/abstraction/chemical.ts`. The `$Chemical` class, `$Synthesis` and `$SynthesisContext`, the `$()` callable, `$check`, `$is`, and `bind`. Composition and the binding constructor. `[SCAFFOLD]`
3. [atom.ts](03-atom.md) — `src/abstraction/atom.ts`. The `$Atom` singleton. The constructor returns the class template; each class has exactly one instance. `[SCAFFOLD]`
4. [bond.ts](04-bond.md) — `src/abstraction/bond.ts`. The `$Bond` class, `$Reagent`, `$Reflection`, decorators (`@inert`, `@reactive`), and the `activate` function. The reactive property installation machinery. `[SCAFFOLD]`
5. [molecule.ts](05-molecule.md) — `src/abstraction/molecule.ts`. The `$Molecule` bond graph collection. Installs getter/setter accessors on instances. Walks the prototype chain to discover bondable properties. `[SCAFFOLD]`
6. [reaction.ts](06-reaction.md) — `src/abstraction/reaction.ts`. The `$Reaction` per-chemical reaction unit. Schedules React state updates. Manages the dirty/clean cycle. `[SCAFFOLD]`
7. [element.ts](07-element.md) — `src/abstraction/element.ts`. React functional component type definitions. The `$Element<T>` and `$Component<T>` types that bridge $Chemistry classes to React's type system. `[SCAFFOLD]`
8. [scope.ts](08-scope.md) — `src/implementation/scope.ts`. Scope tracking and `diffuse`. The `withScope` wrapper, read/write recording, `finalize`, and the fan-out to derivatives. The reactive engine's core. `[SCAFFOLD]`
9. [symbols.ts](09-symbols.md) — `src/implementation/symbols.ts`. Every `Symbol`-keyed slot in the framework. The naming convention (`$x$` for internal, `$$x$$` for framework-private). Why symbols instead of `#private`. `[SCAFFOLD]`
10. [types.ts](10-types.md) — `src/implementation/types.ts`. The TypeScript vocabulary: `$Properties<T>`, `$Component<T>`, `$Element<T>`, `I<T>`. How types survive the `.Component` boundary. `[SCAFFOLD]`
11. [augment.ts](11-augment.md) — `src/implementation/augment.ts`. Event handler wrapping. `augment` walks the React element tree, wrapping handlers in `withScope` so that user interactions trigger scope-tracked reactive updates. `[SCAFFOLD]`
12. [reconcile.ts](12-reconcile.md) — `src/implementation/reconcile.ts`. View diffing. Compares the current view output against the cached previous output. Returns the cached reference when unchanged, triggering React's bailout. `[SCAFFOLD]`
13. [walk.ts](13-walk.md) — `src/implementation/walk.ts`. Tree traversal. The centralized `walk` function that both `augment` and `reconcile` use to traverse React element trees. 39 lines that unify what used to be separate traversals. `[SCAFFOLD]`
14. [representation.ts](14-representation.md) — `src/implementation/representation.ts`. `$symbolize` and `$literalize`. Deterministic serialization for snapshot comparison. Map/Set/Array/Date aware. Cyclic-safe. The foundation of collection mutation detection. `[SCAFFOLD]`
15. [promise.ts](15-promise.md) — `src/implementation/promise.ts`. `$promise` and `$await`. Cancellable promises with chained `then`. The `$cancelled$` sentinel. Synchronous read of settled results. `[SCAFFOLD]`
