# Composition

- **author:** [Cathy](../../../../.claude/library/..teamsmanship/..team/cathy/cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [$Chemistry](../..representivity/.cover.md)

---

I am how chemicals combine. A `$Particle` stands alone; a `$Chemical` composes with others through three modes: **static composition** (JSX inside a render), **binding-constructor composition** (typed children passed to the bond constructor at mount), and **held-instance composition** (`$(instance)` reusing a known instance across renders). The binding constructor is the single most surprising feature in the framework. Every `$Chemical` has two constructors: the class constructor that creates the object, and the binding constructor that receives typed JSX children at render time. The binding constructor is discovered at runtime by name. It is not a method you call — it is a method the framework calls when React renders your children into your component. This book describes composition from the class itself through the dual constructor, the binding protocol, type checking, polymorphism, the catalyst graph that threads parent-child relationships, and the HTML catalogue that adapts native elements into the chemistry system.

## Chapters

1. [The Class](01-the-class.md) — `$Chemical` extends `$Particle`. Adds `$synthesis`, `$catalyst`, `$$parent$$`, `$lastProps$`, `$remove$`. The rules around what composition requires of a chemical.
2. [Dual Constructor](02-dual-constructor.md) — the class constructor (object creation time) vs the binding constructor (render time). Why two moments exist and what each is responsible for. The framework's most counterintuitive design choice.
3. [Binding Constructor](03-binding-constructor.md) — the method named after the class. Discovered at runtime via `chemical[$type$].name`. Receives JSX children as typed arguments. Spread vs positional parameters. The `$Synthesis` orchestrator that parses and dispatches.
4. [Check](04-check.md) — `$check(arg, ...types)`. Runtime parameter validation invoked from inside a binding constructor. Accepts subclass, accepts union, throws on wrong type with a formatted error message designed to be read on a page.
5. [Is](05-is.md) — `$is<T>(ctor)`. Type-only helper for `$check` signatures. The lightweight companion that makes type narrowing explicit.
6. [Bind](06-bind.md) — `bind(chemical, parent?)`. Static binding without JSX. Programmatic composition for cases where the render tree is not the right entry point.
7. [Polymorphism](07-polymorphism.md) — subclass property overrides change appearance without changing parent render code. Polymorphism without props. The mechanism that lets `$VeganRecipe extends $Recipe` look different without the parent knowing.
8. [Catalyst Graph](08-catalyst-graph.md) — `$catalyst$`, `$$parent$$`, `$parent$` setter. How composed chemicals share a reaction system. The rewiring-on-join pattern that threads the parent-child graph through the component tree.
9. [HTML Catalogue](09-html-catalogue.md) — the lazy-memoized `$Html$` wrapper map. The `$('tagname')` and `$('tagname', X)` forms. How native HTML elements enter the chemistry system as adapted chemicals.
10. [Glossary](10-glossary.md) — alphabetical reference of composition terms: `$Chemical`, binding constructor, `$parent$`, `$component$`, `$Atom`, catalyst graph, `$formed$`, `$formation$`, `$remembered$`.
