# Representivity

- **catalogues:** $Chemistry
- **author:** [Libby](../../../../.claude/library/..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **subject:** [$Chemistry](.cover.md)

---

This is the cataloguing book for the $Chemistry branch library. $Chemistry is about representation — how reactive state represents something beyond itself. This branch records the team's knowledge of building that representational framework. Representivity catalogues that knowledge.

## Books

### [Particle](../particle/.cover.md)

Every renderable in $Chemistry inherits from $Particle before the composition layer arrives. This book narrates identity, lifecycle, particularization, per-mount derivation, render filters, the view boundary, and the Component getter — the full surface of the framework's base class. Seven chapters from [Identity](../particle/01-identity.md) through [Component](../particle/07-component.md).

### [Reactivity](../reactivity/.cover.md)

How property writes wake the right consumers. Covers the bond layer, scope tracking, cross-chemical writes, collection mutation, fan-out via `diffuse`, and the decorator overrides that control participation. Six chapters, three with `[SCAFFOLD]` markers for gaps in the source material.

### [Composition](../composition/.cover.md)

How chemicals contain other particles. The dual constructor pattern, the binding constructor that parses typed JSX children, validation, type testing, rebinding, polymorphic dispatch, the catalyst graph, and the HTML catalogue. Nine chapters covering the full chemical layer.

### [Authorship](../authorship/.cover.md)

The component author's daily workflow. Translates the `$` grammar, structural patterns, the export pattern, the reactivity contract, and ecosystem composition into practice. Five chapters for layer-2 readers who subclass `$Chemical` and write `view()`.

### [Implementation](../implementation/.cover.md)

What the source files do. Per-file walkthrough of `library/chemistry/src/`, from `particle.ts` through `promise.ts`. Fifteen chapters, each naming one module, its purpose, and its dependency relationships.

### [Epistemology](../epistemology/.cover.md)

How we know $Chemistry works. Four confirmation surfaces: the Lab, the test suite, caveats, and open questions. Ends with Why $Chemistry — the capstone argument. Six chapters distinguishing speculation from confirmed behavior.

### [Representivity](.cover.md) — $Chemistry

Self-cataloguing. This branch library knowing what it knows about representation.

### [Projection](../projection/.cover.md)

The sprint-by-sprint record of building $Chemistry. Forty sprints from exploration through SSR.
