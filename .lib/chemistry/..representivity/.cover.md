# Representivity

- **catalogues:** $Chemistry
- **author:** [Libby](../../../../.claude/library/..teamsmanship/..team/libby/libby-and-the-tended-garden/.cover.md)
- **subject:** [$Chemistry](.cover.md)

---

This is the cataloguing book for the $Chemistry branch library. $Chemistry is about representation — how reactive state represents something beyond itself. This branch records the team's knowledge of building that representational framework. Representivity catalogues that knowledge.

## Books

### [Particle](../particle/.cover.md)

Every renderable in $Chemistry inherits from $Particle before the composition layer arrives. This book narrates identity, lifecycle, particularization, per-mount derivation, render filters, the view boundary, and the Component getter — the full surface of the framework's base class. Eight chapters from [Identity](../particle/01-identity.md) through [Component](../particle/07-component.md) plus a [glossary](../particle/08-glossary.md) of particle-layer terms.

### [Reactivity](../reactivity/.cover.md)

How property writes wake the right consumers. Covers the bond layer, scope tracking, cross-chemical writes, collection mutation, fan-out via `diffuse`, and the decorator overrides that control participation. Seven chapters including a [glossary](../reactivity/07-glossary.md) of reactivity terms.

### [Composition](../composition/.cover.md)

How chemicals contain other particles. The dual constructor pattern, the binding constructor that parses typed JSX children, validation, type testing, rebinding, polymorphic dispatch, the catalyst graph, and the HTML catalogue. Ten chapters including a [glossary](../composition/10-glossary.md) of composition terms.

### [Authorship](../authorship/.cover.md)

The component author's daily workflow. Translates the `$` grammar, structural patterns, the export pattern, the reactivity contract, and ecosystem composition into practice. Six chapters including a [glossary](../authorship/06-glossary.md) for layer-2 readers who subclass `$Chemical` and write `view()`.

### [Implementation](../implementation/.cover.md)

What the source files do. Per-file walkthrough of `library/chemistry/src/`, from `particle.ts` through `promise.ts`. Fifteen chapters, each naming one module, its purpose, and its dependency relationships.

### [Epistemology](../epistemology/.cover.md)

How we know $Chemistry works. Four confirmation surfaces: the Lab, the test suite, caveats, and open questions. Ends with Why $Chemistry — the capstone argument. Six chapters distinguishing speculation from confirmed behavior.

### [Testing](../testing/.cover.md)

The test suite as specification. 428 tests across 48 files, organized into abstraction, implementation, React integration, regression, specimens, and spikes. What the suite promises, how it's organized, what the framework costs at runtime (the performance contract), how specimens bridge testing and demonstration, and how specific bugs were caught and pinned. Five chapters.

### [Representivity](.cover.md) — $Chemistry

Self-cataloguing. This branch library knowing what it knows about representation.

### [Glossary](01-glossary.md)

Alphabetical index of all $Chemistry vocabulary with one-line definitions and links to the book where each term is defined in context.

### [Projection](../projection/.cover.md)

The sprint-by-sprint record of building $Chemistry. Forty sprints from exploration through SSR.
