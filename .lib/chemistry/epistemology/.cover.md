# Epistemology

- **author:** [Cathy](../../../../.claude/library/..teamsmanship/..team/cathy/cathy-and-the-reactive-canvas/.cover.md)
- **subject:** [$Chemistry](../..representivity/.cover.md)

---

I am how you know whether $Chemistry works. The other books describe what the framework IS and how to use it. This book describes how to verify it, how to question it, and why it exists at all. The epistemic model explains what counts as evidence in this system — not unit test green/red alone, but the Lab application that tests itself by running itself, the test suite that specifies rather than merely asserts, and the caveats that document what was broken and how it was fixed. A framework that formalizes consciousness ought to be honest about what it knows and what it does not. This book is that honesty.

## Chapters

1. [The Epistemic Model](01-the-epistemic-model.md) — what counts as evidence that $Chemistry works. Three levels: the test suite proves contracts, the Lab proves integration, and the sprint record proves the team can build with it. Each level answers a different kind of question.
2. [The Lab](02-the-lab.md) — the `$Chemistry Lab` application. Built front-to-back with the framework it documents. Every page is a chemical. The Lab is continuous integration-by-existence: a bug in the framework is a bug in the Lab.
3. [The Test Suite](03-the-test-suite.md) — 428 tests organized as specifications of what $Chemistry promises. Each test articulates a semantic contract, not an implementation detail. The suite is the normative document; the code is the implementation that satisfies it.
4. [Caveats](04-caveats.md) — resolved bugs and their resolutions. Cross-chemical handler fan-out (sprint 24). Single-letter `$<x>` property names (sprint 24). Particle reactivity allocation (sprint 27). Particularization prototype preservation. Each entry: the historical bug, the fix, the test that pins it.
5. [Open Questions](05-open-questions.md) — provisional behaviors observed in current code that have not been confirmed as the right design. The `parseBondConstructor` regex limits, the unreachable `isViewSymbol` branch, the inherited `$isChemicalBase$` resolution, `$Reagent` reachability. Each entry has the source location, the observed behavior, and the suspected fix.
6. [Why Chemistry](06-why-chemistry.md) — the capstone. The bet against The Good Parts. The four-question pitch: what $Chemistry is, why it exists, its relationship to React, and when to reach for it. A framework that formalizes consciousness exists because someone believed the structure of experience could be implemented, not just described. This chapter restates that belief with forty sprints of evidence behind it.
