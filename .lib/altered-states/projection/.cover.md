The sprint-by-sprint record of the team's effort on altered-states — a psychedelic exploration into altered states of consciousness using mouse visual cortex as our lens. A projection maps one space onto another: sprints live in the team's identity, and Projection maps them onto this branch, recording what the team actually did here in the project's own terms. Each chapter is one sprint — what it aimed to do, what was built, and what was learned — and together they form the branch's autobiography. The last sprint chapter is always the current state. This table of contents is empty at creation and grows as sprints are written. Every sprint gets an entry — created when the plan is known (goal, tasks, owners) and completed at the retro with what was built and learned, per [Sprints](../../../.claude/library/library-tree/03-sprints.md).

## Sprints

1. [Sprint 1 — Does the Mouse Hallucinate?](01-sprint-1--does-the-mouse-hallucinate.md) — set up and retro'd: find evidence for or against DOI-induced hallucination in mouse V1 (thousands of cells, controlled stimuli before/after the psychedelic), and if found, decode what the mouse might see. Built: Nancy onboarded (neuroscientist/philosopher/engineer), the branch scaffolded, [The Altered Cortex](../the-altered-cortex/.cover.md) (dataset verified, seven falsifiable hypotheses, digital-twin/inversion plan), and the analysis **environment** (Python 3.11 + torch/Sensorium + a scan loader, data extracted — task 8 done). The setup and environment phase is complete; the DOI analysis (baseline → altered → evidence → decode, tasks 4–7) awaits execution.
2. [Sprint 2 — Equipping the Team](02-sprint-2--equipping-the-team.md) — the infrastructure that made the team ready to run the analysis, recorded at close: the resumable voice/brain operating model and its documentation, the thinking conventions (think-in-links, bold-only nametags, the brain-as-back-of-mind terminology), a cleaned project library (the neutralized overview deck, a Datasets book, de-voiced papers), the repaired link-checker with links green, and refreshed code assignments / `/responsible`. Enablement, not results — the DOI analysis itself is next.
3. [Sprint 3 — Model-Free First](03-sprint-3--model-free-first.md) — **closed at the descriptive slice**: model-free, but a descriptive first pass before the controlled comparison the plan aimed at — and the pass that proved why that comparison is mandatory. Built: a nine-panel exploration of both sessions and the 749 matched cells (reliability, population rate and co-activity, matched-cell pre/post, behaviour-only drug-state decoding), a shared figure grammar + primitives with a compute cache, and the woven [Initial Data Exploration](../../reports/2026-06-23-exploration/initial-data-exploration.md) with its method book [The Exploration Report](../the-exploration-report/.cover.md). Learned: behaviour alone decodes the session at 97%, so the confound is the precondition for any pre/post claim, not a later phase. Resolved at close: the standard per-scan normalization makes the two scans **comparable** (matched cells on the diagonal, profiles overlapping) — the raw amplitude drop is a per-neuron gain scalar normalization removes, and since the published DOI effect *is* a V1 gain reduction (Michaiel 2019) the magnitude question is deferred to within-scan scale-invariant analysis; the paper was rebuilt accordingly, and the throughline lesson is **research before research**. Rolls forward: the controlled comparison (covariates regressed out), tuning and geometry adapters, the `stats/` module and output tests, and the twin (now **Sprint 4**).
4. [Sprint 4 — The Twin and MEI](04-sprint-4--the-twin-and-mei.md) — **planned**, in a dated experiment (`src/experiments/2026-06-23-twin-mei/`): build aligned Sensorium-baseline twins on **both** scans (post warm-started from pre, so the networks compare), **save the checkpoints durably so a good twin is never lost**, and make the pictures — an MEI grid, a **matched cell's MEI pre vs post**, and **pre-vs-post network visualizations** (first-layer filters, feature maps, the readout receptive-field map). Realizes Step 0–1 of [The Altered Cortex plan](../the-altered-cortex/03-the-analysis-plan.md) and the MEI tool. **Owners: Nancy + Adam build it, Libby catalogues.** Built with the real packages and trained properly — the instrument for Step 0–1, not the controlled H1–H6 tests: a twin that fits is not a mechanism.
5. [Sprint 5 — The publication-grade analysis](05-sprint-5--the-publication-grade-analysis.md) — **closed.** Built the modular `src/analysis/` instrument (data, model, validation, the `verify_twin` harness, `mei_recipe`) done **as the literature does it**, and **resolved the gammas** (6.3831/0.0076 confirmed canonical Lurz/Sensorium — copied, not re-fit). Its **rigorous closing audit — the real worth — caught three bugs before any long run**: the CC_norm ceiling mismatch, the **FEVE survivorship artifact**, and cross-condition cell mis-alignment (now the canonical `matched_pair_index`). Retro: we worked within the library *and* fought it — standing lesson, **convert discipline into structure**. The four-twin run + the analysis carry to Sprint 6.
6. [Sprint 6 — One twin, fully seen, then the rest](06-sprint-6--one-twin-fully-seen.md) — **planned**: the execution sprint under Doug's rule — because the runs are long, **build and fully analyse ONE twin at a time, archive every image and value, look as a team, then decide the next.** Carries Sprint 5's instrument + fixes + the gamma resolution + CD's by-eye verification panel; phases run pre-flight fixes → the full one-twin analysis harness → the first twin fully seen and archived → **the gate** → the rest + the H1–H7 DOI tests on the synced, fixed neuron set. The archive is the inspection record; **Doug drives and inspects — the gate is his call.**
7. [Sprint 7 — The four twins, one at a time](07-sprint-7--the-four-twins-one-at-a-time.md) — ACTIVE — the execution sprint. The gated one-twin loop run via the run_twin launch command: four cases one at a time, starting with behaviour+control (pre/Arm-A, the most standard), as a watchable background process. First twin launched; the by-eye gate caught a y-retinotopy collapse traced to an unregularized shifter (gamma_shifter=0) — a dropped Franke config, sourced from CD before any re-train. Carries Sprint 6 gated rule.
