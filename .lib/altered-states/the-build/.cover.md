How the altered-states analysis is built and run — the worked, verified compute environment, kept apart from the data it loads ([Datasets](../datasets/.cover.md)) and the science it serves ([The Altered Cortex](../the-altered-cortex/.cover.md)). The shape of it: system Python 3.13 is too new for the modelling stack, so a side-by-side **Python 3.11** (fetched with `uv`) carries a **pip-tools-locked venv**; the rock-solid data tier installs clean and is verified first, then the fragile modelling tier — **torch (CPU)** plus the **Sensorium triple** (neuralpredictors, nnfabrik, sensorium) — is wrestled through six specific failures into a lockfile that pins exact versions, **git commit SHAs**, and index URLs. The code is a one-way three-layer split — an importable `src` module core, narrative `notebooks/` that call it, a gitignored `results/` — anchored on a single **frozen digital-twin checkpoint**; and the data's one-`.npy`-per-trial format is disk-bound when cold. The throughline that ties the whole book together: an environment that merely *installs* is not done — an environment that *verifies itself*, every import green, the kernel resolved to the right interpreter, the loader returning the expected shapes, is. This book records how it was built, why each fight happened, and what it takes to rebuild it.
