How the altered-states analysis is built and run — the worked, verified compute environment, kept apart from the data it loads ([Datasets](../datasets/.cover.md)) and the science it serves ([The Altered Cortex](../the-altered-cortex/.cover.md)). The shape of it: system Python 3.13 is too new for the modelling stack, so a side-by-side **Python 3.11** (fetched with `uv`) carries a **pip-tools-locked venv**; the rock-solid data tier installs clean and is verified first, then the fragile modelling tier — **torch (CPU)** plus the **Sensorium triple** (neuralpredictors, nnfabrik, sensorium) — is wrestled through six specific failures into a lockfile that pins exact versions, **git commit SHAs**, and index URLs. The code is a one-way three-layer split — an importable `src` module core, narrative `notebooks/` that call it, a gitignored `results/` — anchored on a single **frozen digital-twin checkpoint**; and the data's one-`.npy`-per-trial format is disk-bound when cold. The throughline that ties the whole book together: an environment that merely *installs* is not done — an environment that *verifies itself*, every import green, the kernel resolved to the right interpreter, the loader returning the expected shapes, is. But the book is not a file-by-file mirror of the tree; it records how the codebase is built, *and how it is allowed to evolve* — the protocols, contracts, and decisions that keep it trustworthy as analyses pile up.

## Coding protocols — how we test and how the code evolves

The conventions the team must remember, surfaced here and specified in **[The coding protocols](09-the-coding-protocols.md)**:

- **The test dial** — `pytest` / `pytest -m core` / `pytest -m <analysis>`, set in [`pyproject.toml`](../../../pyproject.toml): run all, the core, or one analysis.
- **Two kinds of tests** — the shared core is property-tested (`stats/` hardest, every claim inherits its correctness); each analysis **regression-pins** its outputs.
- **The core/volatile contract** — an analysis imports `library/` and **never edits it**; needing the core to change is a **versioned PR with a test, not a quick edit**.
- **Evolution by promotion** — a personal analysis → a project → its own `library/` + `.lib/`; shared code is *moved* up when it earns reuse, never rewritten.
- **The per-task loop** — think → do → **test** → catalogue → review; nothing is done until it is tested and catalogued.
