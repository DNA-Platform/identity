The workspace for the altered-states analysis as a worked environment — how the project is built and run, kept cleanly apart from what it studies ([Datasets](../datasets/.cover.md)) and why it studies it ([The Altered Cortex](../the-altered-cortex/.cover.md)). System Python is 3.13, too new for the modelling stack, so the analysis runs on a **separate Python 3.11** in a **venv** locked with **pip-tools** (`requirements.in` compiled to a fully pinned `requirements.txt`). The heart of the stack is **torch + torchvision (CPU)** beside the **Sensorium triple** — neuralpredictors, nnfabrik, sensorium — pinned to explicit git commits, because the dataset is in exactly that lineage's static-scan format and the digital twin is a torch model; the lighter data work (numpy / scipy / pandas / matplotlib) needs none of it and is split off so exploration is never blocked on the fragile triple. The code sits in three layers — an importable `src` module core, narrative `notebooks/` that *call* it, and a gitignored `results/` — with a single **frozen digital-twin checkpoint** as the most-depended-on node in the graph (provenance-stamped; change it and every downstream number recomputes). This book records the resolved stack and *why*, the module / notebook / artifact split and its one-way dependency, CPU-only execution, and what makes the whole thing reproducible: the data archives immutable, everything else regenerable. Read this to know *how it is built and run*.
